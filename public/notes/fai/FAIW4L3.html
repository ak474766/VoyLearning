1. Introduction to AI Concepts

What is Artificial Intelligence?

Artificial Intelligence (AI) is the science and engineering of making machines behave intelligently. It involves enabling machines to:

  • Perceive their environment - through various sensors and input mechanisms
  • Reason about data - process and analyze information logically
  • Make decisions and learn from experience - adapt and improve over time
Professor mentioned in class: "We are trying to create intelligence artificially, meaning we are trying to mimic human intelligence. Though we have seen that we can go beyond human intelligence in our strong AI part. Think from our perspective - we are also intelligent agents because we are always connecting with the environment and making decisions based on our situation and state at that point of time."

Core areas of AI include:

  • Machine Learning
  • Robotics
  • Natural Language Processing
  • Computer Vision
  • Planning and Decision Making
Professor mentioned in class: "We have a real world environment, and we have an agent which is interacting with this environment through sensors. The agent creates a representation of the real world - like when we want to reach from Jodhpur to Kolkata, we represent this graphically and do all calculations in our mental map before execution."

2.1 What is an Intelligent Agent?

An agent is anything that can perceive its environment through sensors and act upon that environment using actuators.

An intelligent agent acts to achieve the best outcome based on perception, reasoning, and learning.

2.2 Structure of an Intelligent Agent

Every intelligent agent consists of four key components:

Component Function Examples (Biological) Examples (Artificial)
Sensors Input from environment Eyes, ears, touch, smell, taste Camera, microphone, temperature sensor
Actuators Outputs/actions to environment Hands, legs, voice Motors, display, speaker
Agent Function Maps perceptions to actions Brain processing Algorithm/logic
Agent Program Implementation of the function Neural networks Computer code
Professor mentioned in class: "Think about our sensory organs - we have 5 sensory organs that act as sensors. The actuators are the actions we take. The agent function maps perception to actions, and the agent program is the implementation of this function in our brain or in a computer."

2.3 Types of Agents

There are five main types of agents, each with increasing levels of intelligence and capability:

1. Simple Reflex Agents

  • Behavior: Act on current perception only
  • Decision Making: If-then rules based on immediate input
  • Memory: No memory of past states

2. Model-based Reflex Agents

  • Behavior: Keep track of the world using a model
  • Decision Making: Consider current state and internal model
  • Memory: Maintain internal state representation
Professor mentioned in class: "We have to think that our environment is always changing, it is dynamic. So we have to act accordingly. The environment is never static - there are lot of uncertainties, and we have to handle those uncertainties."

3. Goal-based Agents

  • Behavior: Choose actions to achieve a specific goal
  • Decision Making: Planning and search to reach goal state
  • Planning: Find path from initial state to goal state

4. Utility-based Agents

  • Behavior: Choose actions based on utility (happiness/success measure)
  • Decision Making: Optimize for best possible outcome
  • Evaluation: Compare different actions using utility function

5. Learning Agents

  • Behavior: Improve performance over time using feedback
  • Adaptation: Learn from experience and mistakes
  • Evolution: Continuously enhance decision-making ability
Professor mentioned in class: "Every time you are interacting and executing actions, you are getting feedback about how good or bad it was. This feedback helps the agent learn from experience. The learning agent can learn from its experiences and the feedback it gets from executing actions."

4. Agent Environments

The environment in which an agent operates significantly affects its design and performance. Environments can be classified along several dimensions:

4.1 Observable vs Partially Observable Environments

Fully Observable Environment

The agent has access to complete information about the environment state.

  • Chess game - Agent has access to entire board state
  • Classic board games - Checkers, tic-tac-toe
  • Perfect information games - All relevant information is visible

Partially Observable Environment

The agent has limited access to environment information.

  • Self-driving car - Sensors have limited range, can't see everything at once
  • Poker game - Player cannot see opponents' cards
  • Medical diagnosis - Limited patient information available
Professor mentioned in class: "It's not always the case that we can know exactly what our environment is. Like in self-driving cars, we are just looking into a limited portion of the environment because sensors have limited range and cannot see everything at once."

4.2 Deterministic vs Stochastic Environments

Deterministic Environment

Given the same inputs, the outcome is always predictable.

  • Crossword puzzles - Rules are fixed and predictable
  • Calculators - Performing arithmetic operations (2+3 always equals 5)
  • Logic puzzles - Consistent rules and outcomes

Stochastic Environment

Future states involve randomness and uncertainty.

  • Weather prediction - Many uncertain environmental factors
  • Stock market trading - Outcomes depend on many uncertain factors
  • Online gaming - Network latency and player behavior variations
Professor mentioned in class: "In our environment, there are lots of uncertainties. The environment is never static - there's a lot of uncertainty, and we have to handle those uncertainties. Weather and stock market are good examples because they are very volatile with many uncertain factors."

4.3 Episodic vs Sequential Environments

Episodic Environment

Each episode is independent; current actions don't affect future episodes.

  • Image classification - Each image classified independently
  • Spam detection - Each email analyzed separately
  • Quality control - Each product inspected independently

Sequential Environment

Current actions affect future states and decisions.

  • Chess - Current move depends on past moves and influences future moves
  • Robot navigation - Current actions affect future states and positions
  • Conversation systems - Context from previous interactions matters

4.4 Static vs Dynamic Environments

Static Environment

Environment doesn't change while the agent is thinking/deciding.

  • Crossword puzzle - Environment remains same during solving
  • Single-player puzzle games - No time constraints or external changes
  • Mathematical problem solving - Problem doesn't change during calculation

Dynamic Environment

Environment changes continuously, even while agent is deliberating.

  • Self-driving cars - Environment changes continuously (other cars, pedestrians)
  • Real-time strategy games - Multiple players making simultaneous moves
  • Trading systems - Market conditions change rapidly

4.5 Discrete vs Continuous Environments

Discrete Environment

Finite number of states, actions, and time steps.

  • Chess board - Finite number of squares and pieces
  • Turn-based games - Discrete moves and states
  • Digital systems - Finite state machines

Continuous Environment

Infinite range of states, actions, or time (smooth variations).

  • Driving a car - Speed and steering vary continuously
  • Robot arm manipulation - Movement in 3D space
  • Process control - Temperature, pressure variations
