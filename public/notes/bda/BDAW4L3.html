Types of Data File Formats

Data in analytics projects can be stored in various file formats. Each format has unique characteristics, advantages, and challenges. You will most commonly see the following formats:

  • Delimited Text Files
  • Spreadsheets
  • Extensible Markup Language (XML)
  • Portable Document Format (PDF)
  • JavaScript Object Notation (JSON)
Hinglish Summary:
Data analysis mein har format ka apna importance hai. CSV, Excel, XML, PDF, ya JSON—yeh sab commonly use hone wale formats hain jisse data easily store aur share kiya jata hai. Kya format hai, yeh identify karna pehla step hai!
Key Takeaways:
  • Right file format identification saves effort and avoids errors.
  • Formats impact how data is handled, processed, and shared.
  • Each type has some unique pros/cons in analytics tasks.
Practice Questions:
  1. Why is it important to check the file format of a dataset before analysis?
  2. Name two formats suitable for tabular data.
Answers:
  1. File format determines the tools and techniques needed for processing. Using the wrong tool on the wrong format can give wrong results or fail completely.
  2. Delimited text (CSV, TSV) and Spreadsheet (Excel, Google Sheets).

Delimited Text Files (CSV, TSV, etc.)

Delimited Text Files are files where data is stored as plain text and each value is separated by a delimiter—one or more characters that act as a boundary between different data values. The most common types are:

  • CSV: Comma-Separated Values (, as delimiter)
  • TSV: Tab-Separated Values (\t as delimiter)
  • Pipe-separated: | as delimiter

Example from lecture:
facility|outfall|date|bod|nh3|flow|rain|sensor
TX0021211|001|1|A|05/31/2018|3.2|1.42|0.418|3.8|1150

Delimited File Type Delimiter Example Use-case
CSV , Standard for tabular data, spreadsheets
TSV Tab When commas are part of the data entries
Pipe-separated | Specialized datasets, visual distinction

Note: Always check your data for naturally occurring delimiters to avoid parsing errors. For example, using comma as a delimiter when your text values have commas will cause issues.

Spreadsheet Files (Excel/Google Sheets)

Spreadsheet Files organize data into neat rows and columns. Each cell contains a single data value, making it easy to run quick calculations and sort/filter information.

Order # First Name Last Name Country Total Payment Method Status
1 Dalton Kramer France 99 Card In progress
2 Gita Tetterton USA 99 Card Delivered

Spreadsheets often come with functions for statistical analysis—filtering, sorting, analyzing trends, and more. Examples include Microsoft Excel and Google Sheets.

Extensible Markup Language (XML)

XML is a markup language with specific rules for encoding data in a text format, making it easily readable by both humans and machines. Tags in XML define each data point's structure and layout. 

<?xml version="1.0" encoding="UTF-8"?>
<EmployeeData>
  <employee id="34594">
    <firstName>Heather</firstName>
    <lastName>Banks</lastName>
    <hireDate>1/19/1998</hireDate>
    <deptCode>BB001</deptCode>
    <salary>72000</salary>
  </employee>
</EmployeeData>
Advantages Limitations
Platform & language independent, clear structure Verbose, can be storage/processing heavy
Great for data sharing across systems Need parsers/processing software

XML ek tarah ka text format hai jisse data ka layout define hota hai. Machine aur insaan, dono easily padh sakte hai, lekin thoda complex lag sakta hai kyunki tags zyada hote hai. Mostly sharing ke liye use hota hai.
Key Takeaways:
  • XML is ideal for structured data exchange.
  • Its verbose format is both a benefit and a challenge.
Practice Questions:
  1. What is the primary use case of XML?
  2. What distinguishes XML from delimited files?
Answers:
  1. Structured data sharing across platforms and software.
  2. XML uses tags to define data structure; delimited files use simple separators.

Portable Document Format (PDF)

PDF is a widely used format created by Adobe. Its key feature is presentation consistency—the document looks the same on any device or OS.

  • Ideal for invoices, legal/financial documents, government memos.
  • PDFs can also store data gathered from forms (e.g., survey responses).
  • Data extraction from PDFs requires special tools.
[Insert diagram: Example of invoice in PDF structure]
Hinglish Summary:
PDF universal format hai—kisi bhi device ya software pe same dikhai deta hai. Ye mostly official documents ke liye use hota hai, lekin data analysis ke liye pehle extract karna padta hai.
Key Takeaways:
  • PDF is best for presentation, not analysis.
  • Extracting data from PDFs can be tricky and may require OCR or converters.
Practice Questions:
  1. What is the most important feature of a PDF?
  2. Name two scenarios where PDF is used to store data.
Answers:
  1. Visual consistency across all devices and platforms.
  2. Invoices, government memos.

JavaScript Object Notation (JSON)

JSON is a lightweight, text-based format for transmitting structured data over the web. Right now, it's probably the most popular for exchanging data between client and server in applications. Example: 

[
  {"Platform": "Android", "Favorite Food": "Noodle!", "Language": "C#"},
  {"Platform": "iOS", "Favorite Food": "Pasta!", "Language": "Swift"},
  {"Platform": "iOS", "Favorite Food": "Rice!", "Language": "Java"}
]
  • Language independent.
  • Easy to read and write for both humans and machines.
  • Compact and very widely supported in web technologies.
[Insert diagram: JSON structure for data transfer]
Hinglish Summary:
JSON aajkal har app mein use hota hai—chhota, fast, aur sabhi languages read kar sakte hai. Data ko server se client tak bhejne ke liye best hai!
Key Takeaways:
  • JSON is preferred for web-based data exchange.
  • It's compact, flexible and easy for both coding and reading.
Practice Questions:
  1. How does JSON differ from XML?
  2. Where is JSON most commonly used?
Answers:
  1. JSON is more compact, easier to parse, and lacks markup tags; XML uses descriptive tags and is more verbose.
  2. Transmitting data between web servers and web clients/apps.
