Week 10 Lecture 1: Combining Datasets

Date: 2023

Combining Data

  • While working with data, in the data wrangling stage you will often have to combine different datasets.
  • Rarely will your data project involve working with a single dataset.
  • Relational datasets can be put together through various ways.
  • Many firms and companies store important data across different datasets, and to answer research questions, you often need to combine these datasets.

Summary: Data ko analyze karte waqt, aapko aksar multiple datasets ko combine karna padta hai. Real-world projects mein rarely single dataset se kaam chalta hai. Companies apna data alag-alag datasets mein store karti hain, aur research questions ke answers paane ke liye in datasets ko ek saath jodna zaroori hota hai.

Keys

To combine datasets together, we need to understand keys that are used to link datasets:

  • Every join involves a pair of keys.
  • In relational databases, primary keys can be used as fields to join tables on.
  • Keys are the unique information that exists within datasets that allow you to identify records and combine records across datasets.

Primary Key

A variable that uniquely identifies each observation in a dataset.

Compound Key

When more than one variable is needed to uniquely identify an observation.

Foreign Key

A field in a table which references the primary key of another table.

In a relational database, one way to join two tables is by connecting the foreign key from one table to the primary key of another.

Example:

Consider these three related tables:

Table 1: Purchases
Customer ID Product ID Purchase Date
C001 P101 2023-01-15
C002 P102 2023-01-16
C003 P103 2023-01-18
Table 2: Customers
Customer ID Customer Name Payment Mode
C001 Aditya Singh Credit Card
C002 Priya Sharma UPI
C003 Rahul Mehta Debit Card
Table 3: Products
Product ID Product Name Price
P101 Laptop ₹45,000
P102 Smartphone ₹25,000
P103 Headphones ₹2,500

In this example:

  • In the Customers table, Customer ID is the primary key.
  • In the Products table, Product ID is the primary key.
  • In the Purchases table, Customer ID is a foreign key referencing the Customers table.
  • In the Purchases table, Product ID is a foreign key referencing the Products table.
  • The combination of Customer ID and Product ID together could form a compound key in the Purchases table.

Summary: Data tables ko join karne ke liye keys bohot zaroori hoti hain. Primary key ek aisa column hota hai jo har row ko uniquely identify karta hai. Compound key multiple columns ka combination hota hai jo uniquely identify kare. Foreign key ek table mein dusre table ke primary key ko reference karta hai. Foreign key aur primary key ke beech connection banake hum different tables ko join kar sakte hain. Example mein dikhaya gaya hai ki Customer ID aur Product ID kaise different tables ko link karte hain.

Types of Relationships

One-to-One Relationships

When a one-to-one relationship exists between two tables, a given record in one table is uniquely related to exactly one record in another table.

Table A

Record 1

Record 2

Record 3

1:1

Table B

Record A

Record B

Record C

Example:

Table 1: Transactions
Customer ID Purchase Date
C001 2023-01-15
C002 2023-01-16
C003 2023-01-18
Table 2: Customer Information
Customer ID Customer Name Payment Mode
C001 Aditya Singh Credit Card
C002 Priya Sharma UPI
C003 Rahul Mehta Debit Card

In this example, each customer in the first table has exactly one corresponding record in the second table, and vice versa. This is a one-to-one relationship.

One-to-Many Relationships

In a one-to-many relationship, a record in one table can be related to one or more records in a second table. However, a given record in the second table will only be related to one record in the first table.

Table A

Record 1

Record 2

1:N

Table B

Record A (→ 1)

Record B (→ 1)

Record C (→ 2)

Record D (→ 2)

Example:

Table 1: Transactions
Transaction ID Purchase Date
T001 2023-01-15
T002 2023-01-18
Table 2: Transaction Details
Transaction ID Product ID Product Name Price per kg
T001 P101 Rice ₹60
T001 P102 Sugar ₹45
T002 P103 Wheat ₹35
T002 P104 Dal ₹120

In this example, one transaction in the first table corresponds to multiple products in the second table. This represents a hierarchical structure where multiple products can be purchased in a single transaction.

Many-to-One Relationships

This is the reverse of a one-to-many relationship. Multiple records in the first table are related to a single record in the second table.

Table A

Record 1 (→ X)

Record 2 (→ X)

Record 3 (→ Y)

Record 4 (→ Y)

N:1

Table B

Record X

Record Y

Example:

Table 1: Transactions
Transaction ID Customer ID Purchase Date
T001 C001 2023-01-15
T002 C001 2023-01-20
T003 C002 2023-01-18
T004 C002 2023-01-25
Table 2: Customer Information
Customer ID Customer Name Payment Mode
C001 Aditya Singh Credit Card
C002 Priya Sharma UPI

In this example, multiple transactions in the first table are associated with a single customer in the second table. One customer can make multiple purchases on different dates.

Many-to-Many Relationship

In a many-to-many relationship, records in a given table A can be related to one or more records in another table B, and records in table B can also be related to many records in table A.

Table A

Record 1

Record 2

Record 3

N:M

Table B

Record X

Record Y

Record Z

This type of relationship is more rare and complex. If you find yourself dealing with what appears to be a many-to-many relationship, you might be missing some information. It's often resolved by creating a junction table that connects the two tables.

Summary: Tables ke beech 4 types ke relationships ho sakte hain:

1. One-to-One (1:1): Jab Table A ka ek record sirf Table B ke ek hi record se connected ho, aur vice versa. Jaise har customer ka ek unique profile ho.

2. One-to-Many (1:N): Jab Table A ka ek record Table B ke multiple records se connected ho. Jaise ek transaction mein multiple products purchase kiye jaa sakte hain.

3. Many-to-One (N:1): One-to-Many ka ulta. Jab Table A ke multiple records Table B ke ek record se connected hon. Jaise ek customer ke multiple transactions ho sakte hain.

4. Many-to-Many (N:M): Jab Table A ke multiple records Table B ke multiple records se connected hon, aur vice versa. Ye complex hota hai aur usually ek junction table ka use karke resolve kiya jata hai.

Recap

  • Combining datasets is a crucial part of the data wrangling process.
  • Keys (Primary, Compound, Foreign) are essential for identifying and linking records across tables.
  • Different types of relationships can exist between tables:
    • One-to-One (1:1)
    • One-to-Many (1:N)
    • Many-to-One (N:1)
    • Many-to-Many (N:M)
  • In relational databases, you join tables by connecting foreign keys to primary keys.
  • Understanding these concepts is fundamental for effective data analysis across multiple datasets.

Final Summary: Aaj humne seekha ki multiple datasets ko kaise combine kiya jata hai. Keys ka concept samjha - primary key, compound key, aur foreign key. Phir humne different relationship types ke bare mein jana - one-to-one, one-to-many, many-to-one, aur many-to-many. Real-world mein data analysis karte waqt, multiple tables ko join karna zaroori hota hai, aur ye concepts aapko help karenge complex data ko effectively analyze karne mein.
