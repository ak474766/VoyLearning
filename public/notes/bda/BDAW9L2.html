Week 9 - Lecture 2: Errors or Inconsistencies

Data Errors and Inconsistencies

  • After removing duplicates, formatting data and handling missing values, we have to look for errors and inconsistencies in the data
  • This process is more difficult than previous data cleaning steps
  • It's not always obvious which data points are errors
  • Need to use common sense and intuition to check for potential issues

Apne data ko clean karne ke liye duplicates hatane, formatting karne, aur missing values handle karne ke baad, humein data mein errors aur inconsistencies ko dhundhna padta hai. Ye process pehle wale steps se zyada mushkil hota hai kyunki error ko pehchanna hamesha aasan nahi hota. Iske liye humein apne common sense aur intuition ka use karna padta hai.

Identifying Data Errors

  • How would you know if some data was obviously wrong?
  • Need to know the context and source of collection
  • Check for the same information from another source
  • Use domain knowledge to identify impossible or highly improbable values

Example:

In a dataset containing information about individuals, finding a record with:

  • Negative age value (-25): This is clearly an error as age cannot be negative
  • Extremely high age value (132): This is suspicious and likely an error, as it exceeds typical human lifespan

In both cases, you'd need to check the original data source or look for this information in another database to verify.

Data errors ko identify karne ke liye humein context aur data collection ke source ke bare mein pata hona chahiye. Kaise pata chalega ki koi data galat hai? Agar data ke values impossible hain (jaise negative age) ya bahut hi unlikely hain (jaise 132 saal ki age), to ye possible errors ho sakte hain. Aisi situation mein, humein original source check karna chahiye ya same information ko kisi aur jagah se verify karna chahiye.

Data Discrepancies

Data discrepancies occur when the same information appears differently across multiple datasets or databases.

Database 1 Database 2
Name Age Height (inches) Name Age Height (inches)
John 25 72 John 25 72
Mary 31 65 Mary 31 65
Devesh 28 70 Devesh 29 70
Sarah 22 67 Sarah 22 67

Notice how Devesh's age is 28 in Database 1 but 29 in Database 2. This represents a data discrepancy that needs investigation.

Data discrepancies tab hoti hain jab ek hi information alag-alag databases mein differently appear hoti hai. Jaise ki humare example mein, Devesh ki age Database 1 mein 28 hai lekin Database 2 mein 29 hai. Is tarah ki discrepancy ko resolve karne ke liye humein original source ko check karna padega aur decide karna hoga ki konsa data point sahi hai.

Handling Data Errors and Discrepancies

  • Go back to the source to figure out which value is correct
  • Fix any errors you identify by replacing with correct values
  • If unable to determine the correct value:
    • Replace erroneous values with missing values and handle according to missing data strategies
    • May need to drop records completely if errors are severe or widespread
  • Document all changes made to maintain transparency
  • Consider the potential bias introduced when removing records with errors

Data errors aur discrepancies ko handle karne ke liye, sabse pehle original source ko check karke pata karna chahiye ki konsa value sahi hai. Agar sahi value mil jaye, to error ko correct kar dena chahiye. Agar sahi value nahi mil pa rahi hai, to hum us value ko missing value se replace kar sakte hain ya extreme cases mein pure record ko drop kar sakte hain. Lekin yaad rahe, records ko drop karne se bias introduce ho sakta hai, isliye ye decision carefully lena chahiye.

Recap

In this lecture, we covered:

  • The importance of identifying errors and inconsistencies in data after basic cleaning steps
  • Methods to identify potential errors through context, domain knowledge, and cross-referencing
  • How to recognize data discrepancies across multiple databases
  • Strategies for handling data errors:
    • Replacing with correct values when known
    • Converting to missing values when uncertain
    • Dropping records as a last resort

Remember that handling data errors and inconsistencies is the final step in the data cleaning process after removing duplicates, formatting data, and handling missing values.

Is lecture mein humne seekha ki basic data cleaning (duplicates hatana, formatting, missing values) ke baad data mein errors aur inconsistencies ko kaise identify kiya jata hai. Humne dekha ki context, domain knowledge, aur cross-referencing ke through potential errors ko identify kiya ja sakta hai. Data discrepancies ko pehchanne ke tarike aur unhe handle karne ki strategies par bhi charcha ki. Yaad rahe ki data errors ko handle karna data cleaning process ka final step hai.
