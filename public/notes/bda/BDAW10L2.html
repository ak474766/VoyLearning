Joins and Unions

When working with multiple datasets, we often need to combine them to perform comprehensive analysis. This process requires understanding of different types of joins and unions. When joining datasets, not all records may find matching counterparts across datasets. The way we handle these unmatched records determines the type of join we use.

The terminology used in this lecture follows structured querying languages like SQL, but these concepts are consistent across other programming languages like R and Python.

Types of Joins

There are several ways to combine datasets:

  • INNER JOIN
  • LEFT JOIN
  • RIGHT JOIN
  • FULL JOIN
  • UNION

INNER JOIN

An inner join between two tables will return only records where a joining field, such as a key, finds a match in both tables. This is the most common type of join operation.

Example:

Left Table

Transaction ID Purchase Date
1001 2023-01-15
1002 2023-01-18
1003 2023-01-20
1004 2023-01-25
1005 2023-01-30

Results After INNER JOIN

Transaction ID Purchase Date Payment Mode
1001 2023-01-15 Credit Card
1002 2023-01-18 Debit Card
1004 2023-01-25 Cash

Right Table

Transaction ID Payment Mode
1001 Credit Card
1002 Debit Card
1004 Cash
1006 UPI

In this example, only the transactions with IDs 1001, 1002, and 1004 appear in both tables, so only these records are included in the result of the INNER JOIN. Records 1003, 1005, and 1006 are excluded because they don't have matches in both tables.

Summary in Hinglish: Inner Join matlab sirf wahi records aayenge result mein jo dono tables mein match karte hain. Agar koi record ek table mein hai lekin dusre mein nahi, to wo result mein nahi aayega. Ye sabse common join type hai jise hum data science mein use karte hain, jab humen sirf matching records chahiye hote hain.

LEFT JOIN

A left join keeps all of the original records in the left table and returns missing values for any columns from the right table where the joining field did not find a match. As the name suggests, the left table dominates in this type of join.

Example:

Left Table

Transaction ID Purchase Date
1001 2023-01-15
1002 2023-01-18
1003 2023-01-20
1004 2023-01-25
1005 2023-01-30

Results After LEFT JOIN

Transaction ID Purchase Date Payment Mode
1001 2023-01-15 Credit Card
1002 2023-01-18 Debit Card
1003 2023-01-20 NULL
1004 2023-01-25 Cash
1005 2023-01-30 NULL

Right Table

Transaction ID Payment Mode
1001 Credit Card
1002 Debit Card
1004 Cash
1006 UPI

In this example, all records from the left table are included in the result. For transaction IDs 1001, 1002, and 1004, which have matches in the right table, we get complete information. For transaction IDs 1003 and 1005, which don't have matches in the right table, we get NULL values for the Payment Mode column. The record with transaction ID 1006 from the right table is not included at all since it doesn't match any record in the left table.

Summary in Hinglish: Left Join mein hum left table ke saare records rakhte hain, chahe unka right table mein match mile ya na mile. Jahan match nahi milta, wahan right table ke columns mein NULL values aa jati hain. Left table ke records kabhi bhi result se bahar nahi jate, lekin right table ke unmatch records nahi aate result mein. Ye bahut useful hota hai jab aapko left table ka data pure taur par preserve karna ho.

RIGHT JOIN

A right join keeps all of the original records in the right table and returns missing values for any columns from the left table where the joining field did not find a match. Right joins are far less common than left joins, because right joins can always be rewritten as left joins by swapping the table order.

Example:

Left Table

Transaction ID Purchase Date
1001 2023-01-15
1002 2023-01-18
1003 2023-01-20
1004 2023-01-25
1005 2023-01-30

Results After RIGHT JOIN

Transaction ID Purchase Date Payment Mode
1001 2023-01-15 Credit Card
1002 2023-01-18 Debit Card
1004 2023-01-25 Cash
1006 NULL UPI

Right Table

Transaction ID Payment Mode
1001 Credit Card
1002 Debit Card
1004 Cash
1006 UPI

In this example, all records from the right table are included in the result. For transaction IDs 1001, 1002, and 1004, which have matches in the left table, we get complete information. For transaction ID 1006, which doesn't have a match in the left table, we get a NULL value for the Purchase Date column. The records with transaction IDs 1003 and 1005 from the left table are not included at all since they don't match any record in the right table.

Summary in Hinglish: Right Join bilkul Left Join ki ulti operation hai. Ismein right table ke saare records result mein aate hain aur left table se sirf wahi records aate hain jo match karte hain. Jahan match nahi milta, wahan left table ke columns mein NULL values aa jati hain. Asal mein, Right Join ka use kam hota hai kyunki aap tables ko swap karke Left Join mein convert kar sakte hain. Isliye zyada log Left Join hi prefer karte hain.

FULL JOIN

A full join combines a left join and right join. A full join will return all records from both tables, irrespective of whether there is a match on the joining field in the other table, returning null values accordingly.

Example:

Left Table

Transaction ID Purchase Date
1001 2023-01-15
1002 2023-01-18
1003 2023-01-20
1004 2023-01-25
1005 2023-01-30

Results After FULL JOIN

Transaction ID Purchase Date Payment Mode
1001 2023-01-15 Credit Card
1002 2023-01-18 Debit Card
1003 2023-01-20 NULL
1004 2023-01-25 Cash
1005 2023-01-30 NULL
1006 NULL UPI

Right Table

Transaction ID Payment Mode
1001 Credit Card
1002 Debit Card
1004 Cash
1006 UPI

In this example, all records from both tables are included in the result. For transaction IDs 1001, 1002, and 1004, which have matches in both tables, we get complete information. For transaction IDs 1003 and 1005, which are only in the left table, we get NULL values for the Payment Mode column. For transaction ID 1006, which is only in the right table, we get a NULL value for the Purchase Date column.

Summary in Hinglish: Full Join dono Left aur Right Join ka combination hai. Ismein dono tables ke saare records aate hain, chahe unka match ho ya na ho. Jahan match nahi hota, wahan respective columns mein NULL values aa jati hain. Ye tab useful hota hai jab aapko dono tables ka complete data chahiye ho aur aap koi bhi record miss nahi karna chahte, lekin ye operation data redundancy bhi create kar sakta hai.

UNION

The UNION operator is used to vertically combine the results of two statements. For UNION to work without errors, all statements must have the same number of columns and corresponding columns must have the same data type. UNION does not return duplicates.

Example:

Table 1

Transaction ID Purchase Date Payment Mode
1001 2023-01-15 Credit Card
1002 2023-01-18 Debit Card
1003 2023-01-20 UPI

Results After UNION

Transaction ID Purchase Date Payment Mode
1001 2023-01-15 Credit Card
1002 2023-01-18 Debit Card
1003 2023-01-20 UPI
1004 2023-01-25 Cash
1005 2023-01-30 Net Banking
1006 2023-02-05 UPI

Table 2

Transaction ID Purchase Date Payment Mode
1004 2023-01-25 Cash
1005 2023-01-30 Net Banking
1006 2023-02-05 UPI

In this example, UNION combines records from both tables vertically. It's important to note that:

  • Both tables must have the same structure (same number of columns with compatible data types)
  • UNION automatically removes duplicate rows
  • If you want to keep duplicates, you would use UNION ALL (not shown in the example)

Unlike joins, which combine tables horizontally based on matching keys, UNION combines tables vertically by appending rows from one table after another. This is useful when you want to combine results from similar tables or queries.

Summary in Hinglish: UNION operation join ki tarah horizontal combine nahi karti balki vertically data ko jodti hai - matlab ek table ke baad dusri table ke records add ho jaate hain. UNION ke liye zaruri hai ki dono tables mein columns ki sankhya aur unke data types same hon. Ek important baat ye hai ki UNION duplicate records ko automatically remove kar deta hai. Agar aap duplicates rakhna chahte hain to UNION ALL ka use karna padega. Ye operation tab useful hoti hai jab aap similar structure wale alag-alag tables ke data ko combine karna chahte hain.

Recap

We've explored different ways to combine datasets in data analytics:

  • INNER JOIN: Returns only records that have matches in both tables
  • LEFT JOIN: Keeps all records from the left table, returns NULL for non-matching right table records
  • RIGHT JOIN: Keeps all records from the right table, returns NULL for non-matching left table records
  • FULL JOIN: Combines LEFT and RIGHT JOINs, keeping all records from both tables
  • UNION: Combines tables vertically (appends rows), requires same structure, removes duplicates

Understanding these operations is crucial in data science as you'll often need to combine information from multiple tables to answer research questions. Knowing which type of join to use helps you control what records are included in your final dataset and how missing values are handled.

Final Summary in Hinglish: Data science mein, alag-alag tables ko combine karna ek common task hai. Hum ne dekha ki INNER JOIN sirf matching records ko leta hai, LEFT JOIN left table ke saare records rakhta hai, RIGHT JOIN right table ke saare records rakhta hai, FULL JOIN dono tables ke saare records rakhta hai, aur UNION tables ko vertically combine karta hai. In operations ko samajhna bahut zaroori hai kyunki real-world projects mein data aksar kahi jagah se aata hai aur use analyze karne ke liye pehle combine karna padta hai. Correct join type ka chunav karna aapko ye control karne mein madad karta hai ki final dataset mein kaun se records shamil honge aur missing values ko kaise handle kiya jayega.
