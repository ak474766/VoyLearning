1. Introduction to Functions and Procedures

Welcome to this comprehensive exploration of one of the most important and powerful concepts in modern programming: functions and procedures. These concepts form the backbone of professional software development and are central to writing clean, reusable, and maintainable code. As you progress in your journey as a programmer and engineer, understanding how to work with functions will be one of the most valuable skills you can develop.

At its core, a program is a sequence of operations that transforms input data into output data. Every program follows this fundamental pattern: it receives input information, processes that information through a series of steps, and then produces output. While at the machine level, all data is represented as binary values, at our level of abstraction, we work with meaningful representations like numbers, characters, text, images, and videos. Each of these data types comes with its own set of operations that can be performed on them.

Professor mentioned in class: In the previous lectures, we discussed the concept of data types, which are combinations of information and their associated operations. Not every operation is applicable to every data type. For instance, you cannot take the square root of a text string in the same way you do with a number. This specificity is fundamental to understanding how programming languages are structured.

The Layering of Operations

When we think about operations in programming, we can organize them into layers of abstraction. At Level 0, we have the most basic mathematical operators: addition (+), subtraction (-), multiplication (*), and division (/). These are the primitive building blocks of any arithmetic operation.

However, programming becomes truly powerful when we move to Level 1. At this level, we combine two or more basic operations to create more complex operations. For example, we might combine addition and multiplication, or combine subtraction and division to solve more sophisticated problems. This idea of combining simpler operations to create more complex ones is the foundation of procedural programming.

The key insight is that this combination of operations forms a standalone procedure or function. Once we've created this combination, it becomes a reusable mechanism that can be applied to multiple different kinds of problems. This is the power of abstraction in programming.

From Data to Expressions to Programs

Consider a simple example: we have variables x = 3 and y = 4. We can then calculate z = x + y and store the result. Each of these lines represents an operation: the first two are memory operations (storing values), and the third is an arithmetic operation followed by a memory operation (addition and storage). When we combine these operations in sequence, we create a complete program.

In high-level programming languages, every statement or line of code can be evaluated to produce a value or result. This fundamental principle means that a program is essentially a sequence of expressions that build upon each other to solve a problem. Understanding this concept is crucial for moving from writing simple scripts to designing sophisticated, modular programs.

2. Function Calls in Python

Python supports mathematical expressions using what are called function calls. A function call is essentially an invocation of a function with specific arguments. In Python, function calls follow a specific syntax that allows us to use functions in expressions, making our code more readable and powerful.

The Anatomy of a Function Call

A function call in Python has a specific form that consists of three main components: 

name(argument1, argument2, ...)

The components are:

  • Function name: The identifier of the function you want to call (e.g., round, max, abs)
  • Arguments: The values or expressions you pass to the function, separated by commas
  • Parentheses: Used to enclose the arguments and indicate that this is a function call

A critical concept to understand is that arguments are expressions, not just values. This means you can pass not only simple values like numbers, but also entire expressions that Python will evaluate before passing them to the function.

Examples of Function Calls

Let's look at some practical examples of function calls in Python: 

round(2.34)          # Rounds 2.34 to nearest integer
max(a+3, 24)         # Finds maximum of (a+3) and 24
int("42")            # Converts string to integer
float("3.14")        # Converts string to float
bool(1)              # Converts to boolean

In the second example, max(a+3, 24), notice that the first argument is an expression a+3. Python will first evaluate this expression (computing a plus 3), and then pass that result to the max function. This demonstrates the principle that arguments can be any expression.

Functions as Expressions

One of the most important aspects of function calls is that they are expressions. This means they can be used anywhere an expression can appear in Python code. Most commonly, you'll use function calls in assignment statements: 

x = round(2.34)      # Calls round, stores result in x
y = max(10, 20)      # Calls max, stores result in y
z = int("100")       # Calls int, stores result in z

This capability to use function calls as expressions is what makes them so powerful. You can nest function calls, use them in calculations, and combine them with other operations. For example: z = int(round(3.7)) first rounds 3.7 to 4, then converts it to an integer.

Professor mentioned in class: When you use function calls, Python evaluates them from the inside out when they're nested. So in int(round(3.7)), Python first executes round(3.7), gets 4, and then passes that 4 to the int() function. Always remember this order of evaluation!

4. Built-In Functions vs Modules

While Python provides many built-in functions out of the box, there are many more functions that you might expect to find but aren't included as built-ins. This raises an important question: where do these additional functions come from? The answer lies in Python's modular architecture.

Why Are There Limited Built-In Functions?

The designers of Python made a deliberate choice to keep the number of built-in functions relatively small. This is by design! Imagine if every possible function that Python could do was built-in. The language would become incredibly bloated, slow to load, and difficult to maintain. Instead, Python follows a philosophy of keeping the core language lean while providing additional functionality through modules.

For example, you might expect to find functions like sqrt() (square root) or cos() (cosine) among the built-in functions, but they're not. These mathematical functions are instead available through the math module, which you can import when you need them.

What is a Module?

A module is a Python file that contains Python codeâ€”specifically, a collection of functions, variables, and classes that are grouped together by theme or purpose. Modules serve as a way for Python to provide optional functionality. Instead of loading everything at startup, Python loads only the modules you explicitly request through the import statement.

Think of modules as libraries of functionality. Python comes with an extensive standard library containing dozens of modules, each providing specialized functions for different tasks. When you need a specific capability, you import the relevant module, and suddenly you have access to all its functions and variables.

Importing a Module

To access a module, you use Python's import command. The syntax is simple: 

import module_name

Once you've imported a module, you access its functions using the module name as a prefix: 

module_name.function_name(arguments)

This naming convention is important because it prevents namespace conflicts. Different modules might have functions with the same name, and by prefixing with the module name, Python can distinguish between them.

Professor mentioned in class: The prefix requirement might seem like extra typing, but it's actually a feature, not a bug! It makes your code much more readable because anyone reading your code knows exactly which module a function comes from. Professional programmers actually prefer this explicit style.

This module is essential for anyone working with mathematical computations, scientific calculations, or engineering applications. Let's explore how to use it and what it offers.

6. Using Modules and the Import Statement

We've learned that modules are Python files containing useful functions and variables. Now let's dive deeper into the practical aspects of using modules: how to import them, different import strategies, and best practices for working with modules effectively.

The Standard Import Statement

The most common and recommended way to import a module is using the standard import statement: 

import math

When you use this form, you must prefix all functions and variables from the module with the module name. This is actually a good thing because it keeps your code clear and prevents accidental name conflicts. 

>>> import math
>>> math.cos(0)
1.0
>>> cos(0)  # ERROR! NameError: name 'cos' is not defined

In this example, cos(0) doesn't work because Python doesn't know what cos is. You must use math.cos(0) to specify that you want the cosine function from the math module. This explicit notation is a feature that makes your code more readable and maintainable.

Using the "from" Keyword

Python also provides an alternative import mechanism using the from keyword: 

>>> from math import pi
>>> pi
3.141592653589793
>>> from math import *
>>> cos(pi)
-1.0

The first form, from math import pi, imports just the pi constant, and you can use it without the module prefix. The second form, from math import *, imports everything from the math module, and you don't need any prefix for any function or variable.

When NOT to Use "from"

While the from import style might seem convenient, there are important reasons to be cautious with it:

  • Namespace conflicts: Different modules might have functions with the same name. If you import multiple modules using from, you could accidentally overwrite functions or create confusion about which module a function comes from.
  • Code readability: When someone reads your code, they might not immediately know which module a function comes from, making the code harder to understand and debug.
  • Namespace pollution: Using from module import * brings potentially many names into your namespace, making it harder to track what's available.

Professional recommendation: Use the standard import module_name form and prefix your function calls. This is considered best practice in professional Python development.

Other Common Modules

While the math module is extremely useful, Python's standard library includes many other modules for different purposes. Two notable ones are:

  • os module: Provides functions for interacting with the operating system, such as file operations, path manipulation, and accessing environment variables. The os module helps make Python code cross-platform compatible.
  • random module: Provides functions for generating random numbers and selecting random items. You can generate random numbers from different distributions, shuffle sequences, and make random choices for simulations and games.

For example: 

import random
import os

# Get current working directory
current_dir = os.getcwd()

# Generate a random number between 1 and 10
random_num = random.randint(1, 10)

# Randomly choose from a list
colors = ["red", "blue", "green"]
chosen_color = random.choice(colors)

Python's standard library contains about 200+ modules covering everything from file operations to networking to data processing. As you progress, you'll learn to use more of these modules. The key is knowing where to find the documentation when you need a specific capability.

7. Creating Your Own Modules

So far, we've explored Python's built-in functions and the modules provided by the standard library. But one of the most powerful aspects of Python is that you can create your own modules! This allows you to organize your code, make it reusable, and build professional applications. Let's learn how to create a simple module.

Creating a Module: Step by Step

Creating a module is surprisingly simple. A module is just a Python file (with a .py extension) that contains Python code. Let's say you want to create a module called mymodule.py in your project directory. Here's what you would do: 

""" 
A simple module that demonstrates module creation.
This module implements basic mathematical operations.
"""

# Simple variable
x = 1 + 2
x = 3 * x
print(x)

Notice that we've included a docstring at the very beginning of the module. A docstring is a special kind of string (enclosed in triple quotes) that serves as documentation for the entire module. This is a Python convention and best practice that we'll explore in more detail in the next section.

Using Your Custom Module

Once you've created your module file, you can import it just like any other Python module. If your module is in the same directory as your Python script, you can simply import it: 

>>> import mymodule

When you import the module, Python executes all the code in the module file. In our example, this means the code that calculates x = 1 + 2 = 3, then x = 3 * 3 = 9, will be executed when you import the module.

You can then access the variables defined in your module: 

>>> import mymodule
>>> mymodule.x
9

Understanding Module Execution

It's important to understand what happens when you import a module. Consider the following example: 

""" mymodule.py - Module Contents """
x = 1 + 2
x = 3 * x
x  # This line does nothing in a module!

When you import this module and try to access x from the Python shell: 

>>> import mymodule
>>> x
NameError: name 'x' is not defined
>>> mymodule.x
9

The key insight here is that the last line x (just an expression by itself) does nothing when executed in a module. It's only in the interactive Python shell that a bare expression gets printed. In a module, you need to explicitly print values if you want to see them, or access them through the module namespace.

Professor mentioned in class: This is one of the most common sources of confusion for beginning programmers. In the interactive shell, bare expressions are automatically printed. In modules, they're not. That's why we use the print() function in modules when we want to display output. We'll see how to handle this properly when we distinguish between modules and scripts later.

8. Understanding Docstrings

Docstrings are a fundamental part of professional Python code. They're special comments that serve as documentation for your code. Unlike regular comments (which start with #), docstrings are string literals that appear as the first statement in a module, function, class, or method. They're not just ignored by Pythonâ€”they're actually stored and can be accessed programmatically.

What is a Docstring?

A docstring is a string literal enclosed in triple quotes (""") or triple single quotes ('''). It appears as the first statement in a module, function, class, or method. The key difference between docstrings and regular comments is that docstrings are actual string objects that Python preserves and makes accessible, while comments are simply ignored. 

"""
This is a docstring for a module.
It should explain what the module does,
what functions it provides, and how to use it.
"""

# This is a regular comment - it explains something about the code
x = 5

Docstrings in Your Own Modules

When you create a new module, it's essential to include a docstring that describes what your module does. Here's an example: 

"""
My simple module.
This file demonstrates how to create a module with documentation.
Implements simple arithmetic operations.
"""

# Variable definitions
x = 1 + 2
x = 3 * x

This docstring explains the purpose of the module, making it clear to anyone (including yourself months later) what the module is for and what it does. This is much better than having no documentation at all.

Accessing Docstrings with the help() Function

One of the powerful features of docstrings is that they're accessible through Python's help() function. This allows users of your module to get documentation right from the Python shell: 

>>> import mymodule
>>> help(mymodule)

When you run this, Python displays the docstring and other information about the module, making it easy for users to understand what the module provides.

Docstring Conventions (PEP 257)

Python has a style guide called PEP 257 (Python Enhancement Proposal 257) that defines conventions for writing docstrings. Some key conventions include:

  • Docstrings should be concise but complete
  • The first line should be a summary that fits on one line
  • If a longer description is needed, leave a blank line after the summary, then provide more details
  • For module docstrings, describe what the module does and what it provides
  • Use proper grammar and punctuation

Here's an example of a well-formatted module docstring: 

"""
Temperature conversion module.

This module provides functions to convert temperatures between
Celsius, Fahrenheit, and Kelvin scales. It's useful for scientific
calculations and weather-related applications.
"""

Your code might work perfectly, but if no one (including yourself) can understand what it does, it's useless. Always invest time in writing clear, helpful docstrings.

9. User Input in Python

So far, we've learned how to write programs that process data we provide directly in the code. However, in real-world applications, programs need to accept input from users. Whether it's a number to perform a calculation, text to process, or choices to make, understanding how to handle user input is essential for creating interactive programs.

The input() Function

Python provides a built-in function called input() that allows you to get input from the user through the keyboard. The basic syntax is: 

variable = input("Prompt message")

Here's a simple example: 

>>> x = input("Enter a number: ")
Enter a number: 42
>>> x
'42'
>>> type(x)
<class 'str'>

Notice something important: even though we entered the number 42, the input() function returns it as a string! This is one of the most important points to understand about input(). The input() function always returns a string, regardless of what the user types.

Converting Input to the Desired Type

Since input() always returns a string, if you need to work with numbers, you must convert the string to the appropriate type. This is where type casting functions come in handy: 

>>> x = input("Enter a number: ")
Enter a number: 42
>>> x
'42'
>>> y = int(x)
>>> y
42
>>> type(y)
<class 'int'>

In this example, we use int() to convert the string '42' to the integer 42. Similarly, you can use float() to convert to floating-point numbers: 

>>> temperature = input("Enter temperature: ")
Enter temperature: 98.6
>>> temp_float = float(temperature)
>>> temp_float
98.6
>>> type(temp_float)
<class 'float'>

Practical Example: Interactive Program

Let's create a simple interactive program that takes user input and performs calculations: 

import math

# Get radius from user
radius_str = input("Enter the radius of a circle: ")
radius = float(radius_str)

# Calculate area
area = math.pi * radius**2

# Calculate circumference
circumference = 2 * math.pi * radius

# Display results
print(f"Area: {area}")
print(f"Circumference: {circumference}")

This program demonstrates several key concepts: getting user input, converting it to the appropriate type, performing calculations using the math module, and displaying results.

  • Forgetting to convert: Trying to do math with input directly without converting to a number
  • Type mismatch: Trying to convert invalid input (like "hello") to int()
  • Unclear prompts: Providing prompts that don't clearly tell users what to enter

10. Modules vs Scripts

Now that we understand how to create modules and use the input() function, it's important to understand a crucial distinction in Python: the difference between modules and scripts. While they look similar (both are .py files with Python code), they serve very different purposes and should be structured differently.

What is a Module?

A module is a Python file that provides reusable functions, variables, and classes. You import it into other Python programs to access its functionality. A module is designed to be a library or toolkit. The goal is to provide useful code that other programs can use.

Key characteristics of modules:

  • Contains reusable functions and variables
  • Designed to be imported into other programs
  • Should include a docstring explaining what it provides
  • Shouldn't produce output when imported (unless necessary for testing)
  • Focuses on providing utilities and functions
""" temp.py - Temperature conversion module """

def to_fahrenheit(celsius):
    """Convert Celsius to Fahrenheit"""
    return (celsius * 9/5) + 32

def to_celsius(fahrenheit):
    """Convert Fahrenheit to Celsius"""
    return (fahrenheit - 32) * 5/9

When you import this module, you get access to the conversion functions without any output being produced.

What is a Script?

A script is a Python file that behaves like an application or program. It's meant to be run directly, not imported. A script is the final deliverableâ€”the application that the end user runs. The goal is to solve a specific problem or perform specific tasks.

Key characteristics of scripts:

  • Designed to be executed directly (e.g., python myscript.py)
  • May import modules to use their functions
  • Typically produces output or performs actions
  • Contains the main application logic
  • Focuses on solving a specific problem
""" weatherApp.py - Temperature conversion application """
import temp

# Get input from user
celsius_temp = float(input("Enter temperature in Celsius: "))

# Convert and display
fahrenheit_temp = temp.to_fahrenheit(celsius_temp)
print(f"{celsius_temp}Â°C = {fahrenheit_temp}Â°F")

The Key Difference: Print Statements

One of the most important practical differences between modules and scripts is their use of print statements. Let's compare:

Module (temp.py) Script (weatherApp.py)
Contains functions without print Uses print() to display results
Designed for reuse Designed as a finished application
No output when imported Produces output when run
Functions return values Prints values directly

A script uses print() to display output to the user, while a module typically returns values through functions that the calling program can use.

Files Look the Same, But Their Purpose Differs

This is a crucial point: modules and scripts are both .py files and can look identical. The difference is in how they're used and what they're intended to do. Whether a file is a module or a script depends on its purpose and how it's used, not on the file structure itself.

Professor mentioned in class: This is one of the most important concepts to grasp. Don't think "this is a module because of some structural difference." Instead, think: "This is a module because I designed it to provide reusable functions that other programs will import and use. This is a script because I designed it as a complete application that solves a specific problem."

11. Working Directory and Module Location

An important practical aspect of working with modules is understanding how Python locates them. When you try to import a module, Python searches for it in specific locations. The most important of these is your working directoryâ€”the folder where you're currently working.

The Working Directory

The working directory is the folder on your computer where Python looks for files by default. When you run Python or execute a Python script, the working directory is the folder where that script is located or the folder from which you launched Python.

When you try to import a module, Python first looks for it in the working directory. If the module is there, it imports it. If it's not there, Python looks in other system directories and installed packages.

Organizing Your Project Files

Here's a typical project structure: 

my_project/
â”œâ”€â”€ mymodule.py          # Your custom module
â”œâ”€â”€ script.py            # Your main script
â””â”€â”€ data.txt             # Data files

If you want to use mymodule.py in script.py, both files should be in the same directory (my_project). Then, in your script, you can simply: 

import mymodule

The Critical Rule: Working Directory Before Running Python

This is extremely important: You must navigate to the correct working directory before running Python or executing a Python script. If you have a module file but you're not in its directory when you try to run Python, you'll get an ImportError.

For example, if your module is in a folder called code, you need to: 

# Navigate to the directory first
cd code

# Then run Python or your script
python script.py
# or
python

If you try to run Python from a different directory, the import will fail because Python can't find the module.

Checking Your Working Directory

You can check your current working directory using the os module: 

import os
print(os.getcwd())  # Prints the current working directory

The issue is almost always that you're in the wrong working directory. Always double-check that you're in the same directory as your module before trying to import it.
