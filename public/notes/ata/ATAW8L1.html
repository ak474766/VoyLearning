We Write Programs to Do Things

Functions are the key "doers" in our programs - they enable us to encapsulate and reuse computational steps.

Function Call

Command to execute the function

>>> plus(23) 24

Function Definition

Defines what the function does

def plus(n): return n+1

Parameter: Variable listed within parentheses of a method header

Argument: Value assigned to the method parameter when called

Function Encapsulation: Functions allow you to encapsulate (combine) multiple computational steps into one unit that can be reused in repetitive computations. This is one of the most important features of functions.

Program Flow and Function Execution

Programs typically have a sequential execution flow, but functions introduce non-linear execution patterns.

Program Execution Flow

  1. Execution starts from line 1
  2. If a function is defined, it's recorded but not executed
  3. When main program reaches a function call, control jumps to the function definition
  4. After function completes, control returns to the point of call
  5. Execution continues from that point forward

Program Control

Program control determines the flow of logic execution. This is managed by a special register called the Program Counter (PC):

  • PC points to the memory location containing the next line to be executed
  • When a function is called, PC jumps to that function's location
  • When a function returns, PC jumps back to the calling location

Important: When writing programs with functions that call other functions (e.g., a function that calls "add" and "sub"), those called functions should be defined before they are used. Otherwise, the program won't recognize these functions exist.

Function Examples

Example 1: Basic Functions

def sub(a, b): return a - b def mult(a, b): return a * b def add(a, b): return a + b def func(a, b): s = sub(a, b) m = mult(a, b) a = add(a, b) return s, m, a # Main program a = int(input()) b = int(input()) x, y, z = func(a, b) print("Result of subtraction is:", x) print("Result of multiplication is:", y) print("Result of addition is:", z)

This example shows how one function can call other functions and return multiple values.

Variable Scope

Every function creates its own scope (frame) with local variables that don't interfere with variables in other scopes, even if they have the same name.

Example: Local vs Global Scope

def f1(): x = 3 y = 5 print("From function:", x, y) # Main program x = 6 y = 7 print("From main function:", x, y) f1() print("From main function (after calling f1):", x, y)

Output:

From main function: 6 7 From function: 3 5 From main function (after calling f1): 6 7

The variables x and y inside f1() are different from the x and y in the main program.

Understanding Scope

  • Every function creates its own frame (scope)
  • Variables inside a function exist in the function's local scope
  • Variables outside all functions exist in the global scope
  • The same variable name can exist in different scopes without interference
  • When a function completes, its local variables are discarded

Nested Functions

Python allows functions to be defined inside other functions. These are called nested functions.

Example: Nested Functions

def f(x): def g(): x = "abc" print("x is:", x) def h(z): print("z is:", z) x = x + 1 print(x) g() h(x) return x # Main program x = 3 z = f(x) print("x is:", x)

In this example:

  • Function f has two nested functions: g and h
  • These nested functions are created only when f is called
  • Each function has its own scope, even nested ones
  • The x in function g is different from the x in f and the global x

Important: When using nested functions, be careful about variable names. Variables in the outer function's scope can be accessed by inner functions, but they might be shadowed if the inner function defines its own variable with the same name.

Mental Model for Function Execution

When tracing function execution, think about:

  1. Function definitions are recorded but not executed until called
  2. Each function call creates a new frame with its own local variables
  3. Parameters receive copies of the argument values
  4. When a function returns, control goes back to the calling point
  5. The function's frame is discarded, but any return value is preserved

Summary

  • Functions are essential for encapsulating logic and enabling code reuse
  • Program flow is not always sequential - function calls cause jumps in execution
  • Every function creates its own scope with local variables
  • Variables with the same name can exist in different scopes without interference
  • Nested functions are possible and follow the same scoping rules
  • Understanding program control flow is crucial for algorithmic thinking
