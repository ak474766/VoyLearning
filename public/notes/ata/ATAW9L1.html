Structure vs. Flow

In programming, there are two important concepts to understand:

Program Structure

  • Order code is presented
  • How statements are listed
  • Inside/outside of functions
  • Defines possibilities over time

Program Flow

  • Order code is executed
  • Not the same as structure
  • Some statements duplicated
  • Some statements skipped
  • Defines what happens in a single execution

We've already seen this difference with functions. The order in which functions are defined is not necessarily the order in which they are executed. Function execution depends on function calls.

Structure vs. Flow: Example

# Program Structure def foo(): print('Hello') print('World') foo() foo()
# Program Flow > python foo.py World Hello Hello

Important: Bugs often occur when the actual program flow does not match what you expect. Understanding how control flows through your program is crucial for algorithmic thinking.

Conditionals: If-Statements

Conditional statements allow us to break the sequential flow of the program based on certain conditions. They are essential for implementing decision-making in algorithms.

Basic Syntax of If-Statements

if expression: statement(s) # Indented block # Rest of code continues here

Example:

if x > 0: z = x # Put x in z if it is positive

Execution: If the expression x > 0 is True, then the indented statement(s) will be executed. Otherwise, these statements are skipped.

Boolean Expressions

The expression in an if-statement should be a boolean expression - one that evaluates to either True or False.

Logical Operators

Common logical operators used in boolean expressions:

Operator Description Example
< Less than x < y
> Greater than x > y
<= Less than or equal to x <= y
>= Greater than or equal to x >= y
== Equal to x == y
!= Not equal to x != y

Common Mistake: Don't confuse == (comparison) with = (assignment). Using if a = b: is an assignment, not a comparison, and will cause errors.

Note on Non-Boolean Expressions: While best practice is to use boolean expressions in conditionals, Python allows other types of expressions. In these cases:

  • 0, empty strings, None, and empty collections evaluate to False
  • Non-zero numbers, non-empty strings, and non-empty collections evaluate to True

However, using explicit boolean expressions makes your code clearer and helps prevent bugs.

Conditionals: If-Else-Statements

We can extend the if-statement with an else clause to execute alternative code when the condition is False.

Basic Syntax of If-Else-Statements

if expression: statement(s) # Executed if expression is True else: statement(s) # Executed if expression is False # Rest of code continues here

Example: Finding the maximum of two numbers

if x > y: z = x # Put x in z if x is larger else: z = y # Otherwise, put y in z

Execution: If x > y is True, then z = x is executed. Otherwise, z = y is executed.

Control Flow in If-Else Statements

The if-else structure creates a branch point in your program:

           ┌──────┐
           │ Start│
           └──┬───┘
              │
              ▼
        ┌───────────┐
        │Evaluate b │
        └─────┬─────┘
              │
              ▼
        ┌─────────────┐
        │     b?      │
        └─┬─────────┬─┘
          │         │
          ▼         ▼
      ┌───────┐ ┌───────┐
      │   s1  │ │   s2  │
      └───┬───┘ └───┬───┘
          │         │
          │         │
          ▼         ▼
        ┌─────────────┐
        │  Continue   │
        └─────────────┘

Important: For any single execution of the program, only one of the two branches (if or else) will be taken.

Conditionals: If-Elif-Else-Statements

We can chain multiple conditions using elif (else if) when we have more than two possible paths.

Basic Syntax of If-Elif-Else-Statements

if expression1: statement(s) # Executed if expression1 is True elif expression2: statement(s) # Executed if expression1 is False AND expression2 is True elif expression3: statement(s) # Executed if expression1, expression2 are False AND expression3 is True else: statement(s) # Executed if all expressions are False # Rest of code continues here

Example: Finding the maximum of three numbers

if x >= y and x >= z: max_val = x elif y >= x and y >= z: max_val = y else: max_val = z

Important Rules for If-Elif-Else Statements

  • There is no limit to the number of elif statements you can use
  • The elif statements must come after if and before else
  • The else statement is optional
  • Conditions are checked in order, and the first True condition executes its block
  • If an else statement is included, it means "none of the above conditions were True"

Practical Example: Comparing Numbers and Division

Let's examine a complete practical example that demonstrates the use of conditionals to compare numbers and perform division with error handling.

Problem Statement

Take two numbers, see if they are equal, or if one is larger than the other, and also find the result of division.

# Take input from the user x = int(input("Enter first number: ")) y = int(input("Enter second number: ")) # Check if numbers are equal if x == y: print("The input numbers are equal") print("The quotient of division is 1 and remainder is 0") # If not equal, check which one is larger and perform division else: if x > y: print(str(x) + " is greater than " + str(y)) # Check if division is possible (y must not be 0) if y != 0: q = x / y r = x % y print("The quotient of division is " + str(q) + " and remainder is " + str(r)) else: print("Division is not valid because divisor is zero") else: print(str(y) + " is greater than " + str(x)) # Check if division is possible (x must not be 0) if x != 0: q = y / x r = y % x print("The quotient of division is " + str(q) + " and remainder is " + str(r)) else: print("Division is not valid because divisor is zero")

Key Points from the Example

  • We use nested if-else statements to handle different conditions
  • We include error checking for division by zero
  • The program provides appropriate feedback for all possible inputs

Note on Division in Python: By default, the division operator (/) performs floating-point division in Python, returning a float even if the operands are integers. If you want integer division (floor division), use the // operator.

Output Examples:

  • For inputs 4 and 5: "5 is greater than 4" and "The quotient of division is 1.25 and remainder is 1"
  • For inputs 6 and 3: "6 is greater than 3" and "The quotient of division is 2.0 and remainder is 0"
  • For inputs 4 and 0: "4 is greater than 0" and "Division is not valid because divisor is zero"

Conditional Expressions (Ternary Operator)

Python also supports conditional expressions, a compact form of if-else statement:

# Format: value_if_true if condition else value_if_false z = x if x > y else y # Assign the larger of x and y to z

Important: This is an expression, not a statement, so it returns a value.

Key Takeaways

  • Program structure (how code is written) differs from program flow (how code is executed)
  • Conditional statements (if, else, elif) control the flow of execution based on conditions
  • Boolean expressions evaluate to True or False and determine which branch is executed
  • Always consider edge cases (like division by zero) when writing conditional logic
  • Nested conditionals allow for complex decision-making trees
  • Proper indentation is crucial in Python to define the scope of conditional blocks
