Lecture Notes: Program Structure vs. Flow and Control Structures

Introduction

This lecture continues our exploration of algorithmic thinking, building upon the concept of program flow introduced in the previous session. We'll examine how program execution doesn't always proceed linearly—it may jump between different sections of code, reflecting real-world problem-solving patterns where we often need to switch between tasks or revisit previous steps.

Understanding how program flow differs from program structure is fundamental to writing effective code, especially when implementing more complex logic through control structures like conditional statements.

Structure vs. Flow

Program Structure

  • The order in which code is presented
  • How statements are listed in the source file
  • Arrangement of code inside/outside functions
  • Defines the possible paths through the program

Program Flow

  • The order in which code is executed
  • Not the same as structure
  • Some statements may be executed multiple times
  • Some statements may be skipped entirely
  • Defines what happens in a single execution

Important: We've already seen this distinction with functions, where code is defined in one place but execution jumps to the function when called and returns afterward.

Example: Structure vs. Flow

def foo():
    print('Hello')

# Program structure (as written)
foo()
foo()
foo()

When this code is executed, the flow would be:

  1. Enter main program
  2. Encounter first foo() call
  3. Jump to foo() function and execute print('Hello')
  4. Return to main program after first foo() call
  5. Encounter second foo() call
  6. Jump to foo() function and execute print('Hello') again
  7. Return to main program after second foo() call
  8. Encounter third foo() call
  9. Jump to foo() function and execute print('Hello') once more
  10. Return to main program after third foo() call
  11. End program
Hello
Hello
Hello

Note: Programming bugs often occur when the actual flow of execution doesn't match what the programmer expected.

Conditionals: Control Structures

Control structures allow us to break the sequential flow of a program, creating more complex execution paths based on specific conditions. Python provides several conditional structures to control program flow.

If Statements

Format:

if expression:
    statement(s)

If statements allow the program to execute a block of code only if a specified condition (expression) is true. The expression is evaluated to a boolean value (True or False).

Example:

x = int(input())
if x > 0:
    print("This is a positive number")

In this example:

  • If the user inputs a positive number, the message will be printed
  • If the user inputs zero or a negative number, nothing happens

Important: In Python, indentation is crucial as it defines which statements belong to the if block.

If-Else Statements

Format:

if expression:
    statement(s)
else:
    statement(s)

If-else statements provide two alternative paths: one for when the condition is true and another for when it's false.

Example:

x = int(input())
if x > 0:
    print("This is a positive number")
else:
    print("This is a negative number or zero")

In this example:

  • If the user inputs a positive number, the first message will be printed
  • If the user inputs zero or a negative number, the second message will be printed

Control Flow with Conditionals

Flow diagram for if-else statement:

Evaluate Expression
↓
True
↓
Execute if block
False
↓
Execute else block
↓
Continue program

Key point: The program only takes one path for each execution. After the chosen block executes, control returns to the main flow.

Example: Finding Maximum of Two Numbers

def max(x, y):
    if x > y:
        m = x
    else:
        m = y
    return m

# Main program
a = int(input())
b = int(input())
c = max(a, b)
print("Max is:", str(c))

This function compares two values and returns the larger one using an if-else statement.

If-Elif-Else Statements

Format:

if expression1:
    statement(s)
elif expression2:
    statement(s)
elif expression3:
    statement(s)
...
else:
    statement(s)

If-elif-else statements allow for multiple conditions to be checked in sequence. Each elif (else if) provides an additional condition to check if the previous conditions were false.

Example: Finding Maximum of Three Numbers

def max3(x, y, z):
    if x >= y and x >= z:
        return x
    elif y >= x and y >= z:
        return y
    else:
        return z

Rules for elif:

  • There is no limit on the number of elif clauses
  • The elif must be placed between if and else
  • The else clause is optional
  • Conditions are checked in order (top to bottom)
  • The else means that all previous conditions were false

Conditional Expressions (Ternary Operator)

Format:

expression1 if boolean_expression else expression2

Python provides a concise way to write simple if-else statements as expressions. These are often called conditional expressions or ternary operators.

Example:

# Instead of:
if x > y:
    z = x
else:
    z = y

# You can write:
z = x if x > y else y

This is an expression, not a statement, meaning it produces a value that can be used in other expressions or assignments.

Program Flow and Call Frames

When functions are called, the program flow jumps to the function definition and creates a new call frame with local variables. Understanding this process is crucial for tracing program execution.

Example: Variable Tracing in Max Function

def max(x, y):
    if x > y:
        m = x
    else:
        m = y
    return m

# When calling max(0, 3)

Call Frame Execution:

  1. Create a new frame with x=0, y=3
  2. Evaluate condition: x > y (0 > 3) which is False
  3. Skip the if block, execute the else block
  4. Assign m = y (m = 3)
  5. Return m (return 3)
  6. Control returns to the calling point with the value 3

Variable Swapping Example

def max(x, y):
    # Swap if needed so y is always the larger
    if x > y:
        temp = x
        x = y
        y = x
    return y

# Tracing max(3, 0)

Tracing Variables Step by Step:

Step Line x y temp
1 Call max(3, 0) 3 0 -
2 if x > y (3 > 0), True 3 0 -
3 temp = x 3 0 3
4 x = y 0 0 3
5 y = x (should be temp!) 0 0 3
6 return y 0 0 3

Bug spotted! The function has a bug! Line 5 should be y = temp instead of y = x.

This is why tracing through code execution is so important for debugging.

Corrected code:

def max(x, y):
    # Swap if needed so y is always the larger
    if x > y:
        temp = x
        x = y
        y = temp  # Corrected line
    return y

Summary

Key Concepts

  • Program structure and flow are distinct concepts
  • Control structures allow for non-linear program execution
  • Conditional statements (if, if-else, if-elif-else) provide branching
  • Conditional expressions offer concise syntax for simple cases
  • Call frames help track function execution and variable states

Practical Applications

  • Decision-making in algorithms
  • Handling different cases in data processing
  • Validating user input
  • Implementing business logic with multiple conditions
  • Debugging by tracing program flow and variable states
