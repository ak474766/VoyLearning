1. Interactive Pie and Donut Charts

Pie charts and donut charts are circular statistical graphics used to illustrate numerical proportions. They are particularly effective when you want to show how different categories contribute to a whole, such as budget allocations, market shares, or survey results. The key difference between a pie chart and a donut chart is that donut charts have a hole in the center, making them visually distinct and often easier to read when displaying percentages.

Understanding Pie vs Donut Charts

A pie chart is a complete circle divided into slices, where each slice represents a category's proportion of the whole. A donut chart is essentially a pie chart with the center removed, creating a ring shape. This center hole can be used to display additional information or simply to improve visual appeal.

Step-by-Step Implementation

Step 1: Import Required Libraries

We begin by importing the necessary Python libraries. plotly.express provides high-level functions for creating interactive visualizations, while pandas helps us structure our data in DataFrame format.

import plotly.express as px
import pandas as pd
The professor emphasized that Plotly Express (px) is the easiest way to create interactive visualizations. It provides simple, high-level functions that handle most of the complexity for you.

Step 2: Create the Dataset

We create a Python dictionary containing two components: categories and their corresponding budget values. This data structure represents different sectors and their budget allocations.

data = {
    'Category': ['Education', 'Healthcare', 'Infrastructure', 'Defense', 'Others'],
    'Budget': [400, 300, 200, 150, 100]
}

In this example:

  • Education has a budget of 400 crores
  • Healthcare receives 300 crores
  • Infrastructure gets 200 crores
  • Defense is allocated 150 crores
  • Others category receives 100 crores

Step 3: Convert to DataFrame

DataFrames are tabular data structures that make it easy to manipulate and visualize data. We convert our dictionary into a pandas DataFrame:

df = pd.DataFrame(data)

Step 4: Create the Donut Chart

Now we use the px.pie() function to create our interactive visualization. The key parameters are:

Parameter Purpose Description
df Data Source The DataFrame containing our data
names Labels Column name for category labels
values Numerical Data Column name for numerical values
hole Donut Effect Size of center hole (0 = pie chart, 0.4 = donut)
title Chart Title Descriptive title for the visualization 
# Create Donut Chart
fig = px.pie(df, 
             names='Category', 
             values='Budget', 
             hole=0.4,  # This creates the donut hole
             title='Budget Allocation by Category')

# Display the chart
fig.show()

Understanding the 'hole' Parameter

The hole parameter controls the appearance of your chart:

  • hole = 0: Creates a complete pie chart (no hole)
  • hole = 0.4: Creates a donut chart with a moderate-sized hole (recommended)
  • hole = 0.7: Creates a thin ring with a large center hole
The professor demonstrated that when you run this code, Plotly automatically calculates the percentage share of each category. For example, if Education has a budget of 400 out of a total of 1150 (400+300+200+150+100), it will show approximately 34.8% on the chart.

Interactive Features

The generated donut chart is fully interactive. Users can:

  • Hover over any slice to see exact values and percentages
  • Click on legend items to hide/show specific categories
  • Export the chart as a static image (PNG)
  • Zoom and pan (if enabled in settings)

  • Pie charts show proportions as slices of a circle
  • Donut charts are pie charts with a center hole (controlled by hole parameter)
  • px.pie() automatically calculates percentages from values
  • Interactive features include hover information and legend toggling
  • Data must be in DataFrame format before plotting

2. Interactive Geographical Maps (GeoMap)

Geographical maps are powerful visualization tools that display data in the context of physical locations on Earth. Interactive geomaps allow users to explore global or regional data by country, region, or coordinate. They are particularly useful for visualizing economic indicators, population statistics, environmental data, or any information that has a geographical component.

Introduction to GeoMaps

A GeoMap (geographical map) combines location data with statistical information to create meaningful visualizations. In this example, we'll create an interactive world map showing GDP per capita for different countries, with features like:

  • Color-coded continents
  • Bubble sizes representing GDP values
  • Hover information showing country details
  • Rotation and zoom capabilities

Step-by-Step Implementation

Step 1: Import Libraries and Load Dataset

Plotly comes with several built-in datasets for practice and demonstration. The Gapminder dataset contains historical data about countries including life expectancy, population, and GDP.

import plotly.express as px

# Load the Gapminder built-in dataset
df = px.data.gapminder().query("year == 2007")

The .query("year == 2007") method filters the dataset to show only data from the year 2007. This gives us a snapshot of global conditions in that specific year.

The professor explained that Gapminder is a famous dataset used in Hans Rosling's TED talks. It contains comprehensive data about countries from 1952 to 2007. We're filtering for 2007 to get the most recent data in the dataset.

Step 2: Understanding the Dataset Structure

When we examine the filtered dataset, we see 142 rows (countries) and 8 columns:

Column Name Description Example Value
country Country name India, United States
continent Continent name Asia, Europe, Africa
year Year of data 2007 (filtered)
lifeExp Life expectancy in years 65.5, 78.2
pop Total population 1,210,193,422
gdpPercap GDP per capita (USD) 42,951.65
iso_alpha 3-letter country code IND, USA, GBR
iso_num Numerical country code 356, 840

Step 3: Create the Interactive GeoMap

The px.scatter_geo() function creates a scatter plot overlaid on a geographical map. Each country becomes a point on the map, and we can customize various visual attributes:

fig = px.scatter_geo(df, 
                     locations="iso_alpha",      # Country codes
                     color="continent",          # Color by continent
                     hover_name="country",       # Show country name on hover
                     size="gdpPercap",          # Bubble size based on GDP
                     projection="natural earth", # Map projection type
                     title="World GDP per Capita (2007)")

fig.show()

Understanding Key Parameters

1. locations Parameter

The locations parameter specifies how to identify countries on the map. We use "iso_alpha" (ISO 3166-1 alpha-3 codes) which are standardized 3-letter country codes like:

  • IND = India
  • USA = United States
  • GBR = United Kingdom
  • CHN = China

2. color Parameter

Setting color="continent" assigns different colors to each continent, making it easy to identify geographical regions at a glance. Plotly automatically generates a color scheme and legend.

3. hover_name Parameter

This defines what text appears when you hover over a data point. By setting it to "country", the country name will be prominently displayed in the hover tooltip.

4. size Parameter

The size="gdpPercap" parameter makes bubble sizes proportional to GDP per capita. Countries with higher GDP per capita appear as larger circles on the map.

5. projection Parameter

The projection parameter determines how the 3D sphere of Earth is represented on a 2D surface. "natural earth" is a visually appealing projection that balances size and shape distortions. Other options include:

  • orthographic: Globe view
  • mercator: Traditional flat map
  • robinson: Compromise projection
  • equirectangular: Simple latitude-longitude grid
The professor demonstrated that you can rotate the map by clicking and dragging, zoom in on specific regions using the scroll wheel, and double-click to reset the view. The interactive features make it much more engaging than static maps.

Interactive Features of GeoMaps

The generated geographical map provides several interactive capabilities:

  • Rotation: Click and drag to rotate the Earth/map
  • Zoom: Scroll wheel or pinch to zoom in/out
  • Hover Information: Move mouse over countries to see detailed data
  • Legend Filtering: Click continent names in legend to show/hide them
  • Reset View: Double-click to return to original view

3. Interactive 3D Scatter Plot

Three-dimensional scatter plots extend traditional 2D scatter plots by adding a third dimension, allowing us to visualize relationships between three variables simultaneously. This is particularly useful when exploring multidimensional datasets where patterns may not be visible in two dimensions alone.

Understanding 3D Visualization

A 3D scatter plot places data points in a three-dimensional coordinate system with X, Y, and Z axes. Unlike 2D plots that are confined to a flat plane, 3D plots allow you to:

  • Rotate the view to see data from different angles
  • Identify clusters and patterns in three dimensions
  • Add color as a fourth dimension for additional categorization
  • Zoom and pan to explore specific regions

Step-by-Step Implementation

Step 1: Import Required Libraries

import plotly.express as px
import numpy as np
import pandas as pd

numpy is essential for generating random data and performing numerical operations, while pandas helps structure our data.

Step 2: Generate Random 3D Data

To create meaningful 3D visualizations, we first need data with three numerical dimensions. We'll generate random data using NumPy:

# Set random seed for reproducibility
np.random.seed(10)

# Generate 100 random points in 3D space
x = np.random.randn(100)
y = np.random.randn(100)
z = np.random.randn(100)

# Generate random categories for color coding
color = np.random.choice(['A', 'B', 'C'], 100)

Understanding the Data Generation

Random Seed

The np.random.seed(10) function sets a starting point for random number generation. This ensures that every time you run the code, you get the same random numbers, making your results reproducible. The number 10 is arbitrary; you could use any integer.

The professor explained that setting a seed is important for demonstration purposes and debugging. It captures uncertainty while maintaining consistency across multiple runs. In production environments or real experiments, you might omit the seed to get truly random results each time.

Random Normal Distribution

np.random.randn(100) generates 100 random numbers from a standard normal distribution (mean = 0, standard deviation = 1). This creates a bell-curved distribution of values, mostly clustering around zero.

Random Choices for Categories

np.random.choice(['A', 'B', 'C'], 100) randomly selects 100 items from the list ['A', 'B', 'C']. This creates categorical data that we'll use for color coding our 3D points.

Step 3: Create DataFrame

We organize our generated data into a structured DataFrame:

df = pd.DataFrame({
    'x': x,
    'y': y,
    'z': z,
    'color': color
})

This creates a table with four columns: x, y, z (coordinates), and color (category).

Step 4: Create the 3D Scatter Plot

fig = px.scatter_3d(df, 
                    x='x',           # X-axis column
                    y='y',           # Y-axis column
                    z='z',           # Z-axis column
                    color='color',   # Color by category
                    title='Interactive 3D Scatter Plot')

fig.show()

Understanding Key Parameters

Parameter Purpose Description
df Data Source The DataFrame containing all data
x X-axis Column name for horizontal axis
y Y-axis Column name for vertical axis
z Z-axis (Depth) Column name for depth dimension
color Categorization Column for color coding (4th dimension)
title Chart Title Descriptive title for the plot

Interactive Features of 3D Scatter Plots

The generated 3D scatter plot is highly interactive with the following capabilities:

1. Rotation

Click and drag anywhere on the plot to rotate the 3D space. This allows you to view your data from different angles, revealing patterns that might be hidden from a single viewpoint.

2. Zoom

Use your mouse scroll wheel (or pinch gesture on touchscreens) to zoom in and out. This helps you focus on specific clusters or examine outliers more closely.

3. Pan

Hold Shift and drag to pan (move) the plot without rotating it. This is useful when you've zoomed in and want to explore different regions.

4. Hover Information

Move your mouse over any data point to see its exact X, Y, Z coordinates and category information in a tooltip.

5. Reset View

Double-click anywhere on the plot to reset to the original view orientation.

The professor demonstrated that you can rotate the plot left, right, up, and down to see the data from all angles. He emphasized that this interactivity is what makes 3D plots in Plotly superior to static 3D visualizations, as it allows for true exploration of the data space.

Practical Applications

3D scatter plots are particularly useful in fields such as:

  • Machine Learning: Visualizing feature spaces with three principal components
  • Physics: Plotting particle positions or trajectories in 3D space
  • Chemistry: Molecular structure visualization
  • Economics: Analyzing relationships between three economic indicators
  • Biology: Plotting morphological measurements across three dimensions

4. Animated Scatter Plot (Time-series Motion)

Animated scatter plots bring the dimension of time into data visualization, allowing viewers to see how data evolves over time. This is particularly powerful for understanding trends, changes, and patterns in temporal data. The most famous example is Hans Rosling's presentation showing how countries developed over decades using animated bubble charts.

Understanding Time-Series Animation

An animated scatter plot is essentially a series of scatter plots shown in sequence, where each frame represents a different time period. As the animation plays, you can observe:

  • How individual data points move across the plot over time
  • Emerging trends and patterns in the data
  • Relative changes between different groups or categories
  • Convergence or divergence of data clusters

Step-by-Step Implementation

Step 1: Import Libraries and Load Dataset

import plotly.express as px

# Load complete Gapminder dataset (all years)
df = px.data.gapminder()

Unlike our previous GeoMap example where we filtered for a single year, here we load the complete Gapminder dataset containing data from multiple years (1952-2007). This temporal data is essential for creating animation.

Dataset Overview: The complete Gapminder dataset contains 1,704 rows representing country-year combinations from 1952 to 2007, measured every 5 years. This gives us 12 time points for each country.

Step 2: Create the Animated Scatter Plot

Now we'll create an animated visualization showing global development over time:

fig = px.scatter(df, 
                 x="gdpPercap",              # X-axis: GDP per capita
                 y="lifeExp",                # Y-axis: Life expectancy
                 animation_frame="year",     # Animate over years
                 animation_group="country",  # Track countries across frames
                 size="pop",                 # Bubble size by population
                 color="continent",          # Color by continent
                 hover_name="country",       # Show country on hover
                 log_x=True,                 # Logarithmic X-axis
                 size_max=60,                # Maximum bubble size
                 range_x=[100, 100000],      # X-axis range
                 range_y=[20, 90],           # Y-axis range (life expectancy)
                 title='Global Development Over Time')

fig.show()

Understanding Key Parameters

Animation Parameters

Parameter Purpose Description
animation_frame Time Dimension Column that defines animation frames (here: "year")
animation_group Tracking ID Column that tracks entities across frames (here: "country")
The professor emphasized that animation_frame tells Plotly which column to use for creating different frames of the animation, while animation_group ensures that the same country is tracked consistently as it moves across the plot over time. Without animation_group, you might see countries disappearing and reappearing rather than smoothly transitioning.

Visual Parameters

Parameter Value/Purpose Explanation
x="gdpPercap" Horizontal axis GDP per capita (economic indicator)
y="lifeExp" Vertical axis Life expectancy (health indicator)
size="pop" Bubble size Population (larger bubbles = more people)
color="continent" Color coding Different colors for each continent
hover_name="country" Tooltip label Country name appears on hover

Scale and Range Parameters

Logarithmic X-axis (log_x=True): GDP per capita varies enormously between countries‚Äîfrom hundreds to tens of thousands of dollars. A logarithmic scale compresses this wide range, making it easier to compare countries across different economic levels. On a log scale, the distance from 1,000 to 10,000 is the same as from 10,000 to 100,000.

Maximum Bubble Size (size_max=60): This limits the maximum size of bubbles, preventing extremely populous countries (like China and India) from overwhelming the visualization.

Range Specifications:

  • range_x=[100, 100000]: Sets GDP per capita range from $100 to $100,000
  • range_y=[20, 90]: Sets life expectancy range from 20 to 90 years

These fixed ranges ensure that the plot axes don't change as the animation plays, making it easier to track movement and changes over time.

Interpreting the Animation

What the Plot Shows

This animated visualization, inspired by Hans Rosling's famous presentations, reveals several global development trends:

  • Economic Growth: Countries generally move rightward (increasing GDP per capita)
  • Health Improvements: Countries move upward (increasing life expectancy)
  • Population Growth: Bubbles grow larger over time (particularly China and India)
  • Regional Patterns: Different continents cluster in different regions of the plot
The professor pointed out that when you play the animation from 1952 to 2007, you can clearly see two large bubbles (China and India) because of their massive populations. You can also observe how most countries move toward the upper-right corner of the plot, indicating improvements in both wealth and health. Some African countries, however, show stagnation or decline in certain periods, particularly during the HIV/AIDS crisis of the 1990s.

Interactive Controls

The animated scatter plot provides several interactive controls:

1. Play/Pause Button

Click the play button to start the animation. It automatically cycles through all years from 1952 to 2007. Click pause to stop at any point.

2. Year Slider

Drag the slider to jump to specific years or manually step through the animation frame by frame.

3. Legend Filtering

Click on continent names in the legend to show/hide specific continents. For example, if you only want to see Asian countries' development, you can hide other continents.

4. Hover Information

At any point during or after the animation, hover over bubbles to see detailed information: country name, exact GDP per capita, life expectancy, population, and year.

5. Zoom and Pan

Even during animation, you can zoom into specific regions of the plot to examine particular groups of countries more closely.

5. Interactive Heatmap

A heatmap is a data visualization technique that displays values in a matrix format using color gradients. The intensity of color represents the magnitude of values‚Äîdarker or brighter colors indicate higher or lower values. Heatmaps are excellent for identifying patterns, correlations, and anomalies in tabular data at a glance.

Understanding Heatmaps

Heatmaps are particularly useful when you have:

  • Matrix Data: Data organized in rows and columns
  • Correlation Matrices: Showing relationships between variables
  • Time-Series Data: Values changing over time periods
  • Geographical Data: Like weather patterns across regions
  • Performance Metrics: Comparing multiple entities across multiple criteria
The professor mentioned that we commonly see heatmaps in weather reporting, where different colors represent temperature or rainfall variations across geographical locations. For example, red might indicate high temperatures while blue indicates cooler areas.

Step-by-Step Implementation

Step 1: Import Required Libraries

import plotly.express as px
import numpy as np
import pandas as pd

Step 2: Generate Random Data

For demonstration purposes, we'll create a 10√ó10 matrix of random values:

# Generate random data: 10 rows √ó 10 columns
data = np.random.rand(10, 10)

np.random.rand(10, 10) creates a 10√ó10 matrix filled with random values between 0 and 1. Each cell contains a random decimal number that we'll visualize using color intensity.

Step 3: Create DataFrame with Column Names

To make our heatmap more interpretable, we'll assign meaningful column names:

# Create DataFrame with column names C0, C1, C2, ..., C9
df = pd.DataFrame(data, columns=[f'C{i}' for i in range(10)])

The list comprehension [f'C{i}' for i in range(10)] creates column names: C0, C1, C2, C3, C4, C5, C6, C7, C8, C9. The f'C{i}' is an f-string that formats the column name dynamically.

Understanding List Comprehension: The expression [f'C{i}' for i in range(10)] is a concise way to create a list. It's equivalent to: 
columns = []
for i in range(10):
    columns.append(f'C{i}')

Step 4: Create the Interactive Heatmap

Now we use Plotly's imshow() function (image show) to create the heatmap:

fig = px.imshow(df, 
                color_continuous_scale='viridis',
                title='Interactive Heatmap Example')

fig.show()

Understanding Key Parameters

The imshow() Function

px.imshow() stands for "image show" and is designed to display matrix data as an image. Each cell in the matrix becomes a colored square in the heatmap, with color representing the cell's value.

Color Scales

The color_continuous_scale parameter determines the color scheme used in the heatmap. Viridis is a perceptually uniform color scale that works well for most data. It ranges from dark purple (low values) through blue and green to bright yellow (high values).

Color Scale Description Best Used For
viridis Purple ‚Üí Blue ‚Üí Green ‚Üí Yellow General purpose, colorblind-friendly
plasma Purple ‚Üí Pink ‚Üí Orange ‚Üí Yellow High contrast visualizations
blues Light Blue ‚Üí Dark Blue Single-hue representations
rdbu Red ‚Üí White ‚Üí Blue Diverging data (positive/negative)
hot Black ‚Üí Red ‚Üí Yellow ‚Üí White Temperature-related data

Reading the Heatmap

In the generated heatmap:

  • Columns are labeled C0 through C9 (horizontal axis)
  • Rows are numbered 0 through 9 (vertical axis)

6. Dashboard-style Subplots

Dashboards combine multiple visualizations in a single view, providing a comprehensive overview of data from different perspectives. Instead of creating separate plots, subplots arrange multiple charts in a grid layout, making it easy to compare and analyze related information side-by-side. This is the hallmark of professional data presentations and business intelligence tools.

Understanding Subplots

A subplot is a plot that exists within a larger figure containing multiple plots. Think of it as dividing your canvas into a grid (rows and columns), and placing different visualizations in each cell. This approach is particularly valuable when:

  • You want to show multiple related visualizations together
  • You need to compare different aspects of the same data
  • You're creating a comprehensive dashboard or report
  • You want to save space by combining multiple charts

Step-by-Step Implementation

Step 1: Import Required Libraries

from plotly.subplots import make_subplots
import plotly.graph_objects as go
import numpy as np

Here we introduce plotly.graph_objects (imported as go), which provides low-level control over plot elements. We also import make_subplots, a specialized function for creating subplot grids.

Plotly Express vs Graph Objects: While plotly.express (px) provides high-level, easy-to-use functions, plotly.graph_objects (go) offers more granular control. For complex layouts like subplots, we often use both together.

Step 2: Generate Data for Multiple Plots

We'll create sample data representing different mathematical functions:

# Generate x values from 0 to 10
x = np.linspace(0, 10, 100)

# Generate different y functions
y1 = np.sin(x)           # Sine wave
y2 = np.cos(x)           # Cosine wave
y3 = np.exp(-x)          # Exponential decay

Understanding the Data Generation

np.linspace() Function

np.linspace(0, 10, 100) creates 100 evenly spaced values between 0 and 10. Unlike range() which only works with integers, linspace creates floating-point numbers with equal spacing: \[0, 0.101, 0.202, ..., 9.898, 10.0\]

Mathematical Functions

  • Sine Function: $y_1 = \sin(x)$ - Creates a smooth oscillating wave between -1 and 1
  • Cosine Function: $y_2 = \cos(x)$ - Similar to sine but shifted by $\pi/2$ radians
  • Exponential Decay: $y_3 = e^{-x}$ - Starts at 1 and rapidly decreases toward 0

Step 3: Create the Subplot Grid

# Create a 2√ó2 grid of subplots
fig = make_subplots(rows=2, cols=2,
                    subplot_titles=('Sine Wave', 'Cosine Wave', 
                                  'Exponential Decay', 'Combined View'))

The make_subplots() function creates a figure with a grid layout:

  • rows=2, cols=2: Creates a 2√ó2 grid (4 plots total)
  • subplot_titles: A tuple containing titles for each subplot
Position Row, Col Title
Top-left Row 1, Col 1 Sine Wave
Top-right Row 1, Col 2 Cosine Wave
Bottom-left Row 2, Col 1 Exponential Decay
Bottom-right Row 2, Col 2 Combined View

Step 4: Add Traces to Subplots

Now we add individual plots (called "traces") to specific positions in the grid:

# Add trace to Row 1, Column 1 (Sine)
fig.add_trace(go.Scatter(x=x, y=y1, mode='lines', name='sin(x)'), 
              row=1, col=1)

# Add trace to Row 1, Column 2 (Cosine)
fig.add_trace(go.Scatter(x=x, y=y2, mode='lines', name='cos(x)'), 
              row=1, col=2)

# Add trace to Row 2, Column 1 (Exponential)
fig.add_trace(go.Scatter(x=x, y=y3, mode='lines', name='exp(-x)'), 
              row=2, col=1)

# Add trace to Row 2, Column 2 (Combined)
fig.add_trace(go.Scatter(x=x, y=y1+y2, mode='lines', name='sin+cos'), 
              row=2, col=2)

Understanding go.Scatter()

The go.Scatter() function creates a scatter/line plot with these parameters:

  • x: Data for x-axis
  • y: Data for y-axis
  • mode='lines': Display as connected lines (other options: 'markers', 'lines+markers')
  • name: Label for the legend

The row and col parameters specify where in the grid to place this trace.

The professor emphasized that in the fourth subplot (row 2, column 2), we're showing a combined view by plotting y1+y2, which adds the sine and cosine functions together. This demonstrates how multiple mathematical functions can be combined and visualized.

Step 5: Update Layout and Display

# Update overall figure layout
fig.update_layout(height=700, width=900, 
                  title_text="Interactive Subplot Dashboard")

# Display the figure
fig.show()

The update_layout() method sets overall properties:

  • height=700: Figure height in pixels
  • width=900: Figure width in pixels
  • title_text: Main title above all subplots

Interactive Features of Subplots

Each subplot in the dashboard is fully interactive:

1. Independent Zoom

You can zoom into any individual subplot by selecting a region with your mouse. Each subplot maintains its own zoom level independently.

2. Unified Legend

A single legend shows all traces. Clicking a legend item hides/shows that trace across all relevant subplots.

3. Hover Information

Hover over any line in any subplot to see exact coordinate values.

4. Double-Click Reset

Double-clicking on a subplot resets its zoom to the original view.

The professor demonstrated that you can select any area of a plot and zoom in, then double-click to zoom back out. This makes it easy to focus on specific regions of interest in any subplot while maintaining context from the other plots.

Practical Applications

Dashboard-style subplots are extensively used in:

  • Business Intelligence: Showing multiple KPIs (Key Performance Indicators) together
  • Scientific Research: Comparing experimental results across different conditions
  • Financial Analysis: Displaying stock prices, volumes, and technical indicators
  • Engineering: Showing multiple sensor readings or system metrics simultaneously
  • Data Science: Feature distribution analysis and model comparison

7. Export and Share Interactive Plots

One of the most powerful features of Plotly is the ability to export your interactive visualizations as standalone HTML files. This means you can share your plots with others, and they'll be able to interact with them (hover, zoom, pan) even without Python or Plotly installed. This is invaluable for reports, presentations, and web applications.

Why Export to HTML?

Exporting to HTML format provides several advantages:

  • Full Interactivity Preserved: All hover effects, zoom, and pan features remain functional
  • Standalone Files: No dependencies required‚Äîjust open in a web browser
  • Easy Sharing: Send via email, upload to websites, or embed in web applications
  • Cross-Platform: Works on any device with a web browser (desktop, mobile, tablet)
  • Lightweight: HTML files are typically small and easy to distribute

Exporting Your Plots

Basic Export Syntax

To export any Plotly figure to HTML, use the write_html() method:

# Export to HTML file
fig.write_html("interactive_plot.html")

This creates a file named interactive_plot.html in your current working directory. You can then:

  • Open it directly in any web browser
  • Share it with colleagues via email or cloud storage
  • Embed it in websites or documentation
  • Include it in reports or presentations

Complete Export Examples

Example 1: Export a Pie Chart

import plotly.express as px
import pandas as pd

data = {'Category': ['A', 'B', 'C'], 'Values': [30, 50, 20]}
df = pd.DataFrame(data)
fig = px.pie(df, names='Category', values='Values')

# Export to HTML
fig.write_html("pie_chart.html")

Example 2: Export a Dashboard

from plotly.subplots import make_subplots
import plotly.graph_objects as go

fig = make_subplots(rows=1, cols=2)
fig.add_trace(go.Scatter(x=[1,2,3], y=[4,5,6]), row=1, col=1)
fig.add_trace(go.Bar(x=[1,2,3], y=[7,8,9]), row=1, col=2)

# Export dashboard
fig.write_html("dashboard.html")

Advanced Export Options

1. Customizing File Location

You can specify a full path:

fig.write_html("/content/drive/MyDrive/plots/my_plot.html")

2. Include Plotly.js Locally

By default, the HTML file references Plotly.js from a CDN. To make it fully self-contained:

fig.write_html("plot.html", include_plotlyjs='cdn')  # Default (requires internet)
fig.write_html("plot.html", include_plotlyjs=True)   # Includes library (larger file)

3. Auto-open in Browser

fig.write_html("plot.html", auto_open=True)  # Opens automatically in default browser

4. Configuration Options

config = {'displayModeBar': False}  # Hide the toolbar
fig.write_html("plot.html", config=config)

Other Export Formats

Plotly also supports exporting to static image formats:

PNG Export

fig.write_image("plot.png")  # Requires kaleido library

PDF Export

fig.write_image("plot.pdf")

SVG Export

fig.write_image("plot.svg")  # Vector format (scalable)
Note: Static image export (PNG, PDF, SVG) requires the kaleido library. Install it with: pip install kaleido

Sharing Best Practices

  • File Naming: Use descriptive names like sales_2024_dashboard.html

  • Privacy: Ensure data is anonymized if sharing sensitive information
  • Version Control: Keep track of different versions if your data updates regularly
