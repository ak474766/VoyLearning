1. Introduction to Subplots

What are Subplots?

Subplots are a way to display multiple plots within a single figure. Instead of creating separate figures for each visualization, subplots allow you to organize and present multiple related plots together on one canvas, making it easier to compare and analyze different datasets or aspects of the same data.

üéì Professor mentioned in class:

"Think of your plotting canvas as being divided into sections - like dividing a paper into different parts. If we want to show multiple plots, we can use subplots to place particular plots at particular positions. Say you want to show 4 plots, so you can divide your complete canvas into 4 parts and then for each part you can show different plots."

The beauty of subplots lies in their flexibility. You have various options for how these plots should be arranged:

  • Horizontal arrangement: Plots placed side by side in a row
  • Vertical arrangement: Plots stacked on top of each other in a column
  • Grid arrangement: Plots organized in a matrix format (e.g., 2√ó2, 3√ó2, etc.)
  • Shared axes: Multiple plots sharing a common X-axis or Y-axis

When to Use Subplots?

Subplots are particularly useful in the following scenarios:

Use Case Description Example
Comparing Multiple Datasets When you need to visually compare different datasets side by side Comparing sales data across different regions
Showing Related Metrics Displaying different metrics or aspects of the same data Temperature, humidity, and pressure readings over time
Before/After Comparisons Visualizing data transformations or processing steps Original image vs. filtered image
Multiple Views of Data Different visualizations of the same dataset Scatter plot, histogram, and box plot of the same data
Time Series Analysis Displaying multiple time-dependent variables together Stock prices of different companies over the same period

üí° Key Advantage

When subplots are arranged horizontally, they can share a common Y-axis, making it easier to compare values. Similarly, vertically arranged subplots can share a common X-axis, which is especially useful for time-series data where you want to align events across different metrics.

2. Subplots with Matplotlib

Matplotlib provides several methods for creating subplots. In this section, we'll explore the most common approaches, from basic to advanced techniques.

Basic Subplots using plt.subplot()

The plt.subplot() function is the traditional way to create subplots. It allows you to specify the grid layout and position for each subplot individually.

Syntax

The basic syntax is:

plt.subplot(nrows, ncols, index)
div>
Parameter Description Example
nrows Number of rows in the subplot grid 1, 2, 3...
ncols Number of columns in the subplot grid 1, 2, 3...
index Position of the current subplot (1-indexed) 1, 2, 3... (starts from 1)

‚ö†Ô∏è Important Note

In plt.subplot(), the indexing starts from 1, not 0. The subplots are numbered from left to right, top to bottom.

Complete Example: Sine and Cosine Waves

üéì Professor's Walkthrough:

"Let me walk you through this code step by step. We'll create two subplots - one for sine wave and one for cosine wave. First, we'll import the necessary libraries, then generate data, and finally create the subplots horizontally - one row with two columns."


# Step 1: Import necessary libraries 
import matplotlib.pyplot as plt import numpy as np

# Step 2: Generate data for plotting 
# Create 100 evenly spaced points between 0 and 10 
x = np.linspace(0, 10, 100)

# Calculate sine values for each x point 
y1 = np.sin(x)

# Calculate cosine values for each x point 
y2 = np.cos(x)

# Step 3: Create a figure with specified size 
# figsize=(width, height) in inches 
plt.figure(figsize=(10, 4))

# Step 4: Create first subplot (1 row, 2 columns, position 1) 
plt.subplot(1, 2, 1) 
plt.plot(x, y1, 'b', label='sin(x)') 

# 'b' = blue color 
plt.title('Sine Wave') 
plt.xlabel('x') 
plt.ylabel('sin(x)') 
plt.legend()

# Step 5: Create second subplot (1 row, 2 columns, position 2) 
plt.subplot(1, 2, 2) 
plt.plot(x, y2, 'r', label='cos(x)') 
# 'r' = red color 
plt.title('Cosine Wave') 
plt.xlabel('x') 
plt.ylabel('cos(x)') 
plt.legend()

# Step 6: Adjust layout to prevent overlapping 
plt.tight_layout()

# Step 7: Display the figure 
plt.show()

Code Explanation

  1. Library Import: We import matplotlib.pyplot for plotting functions and numpy for numerical operations
  2. Data Generation:
    • np.linspace(0, 10, 100) creates 100 equally spaced points from 0 to 10
    • We calculate sine and cosine values using NumPy's built-in functions
  3. Figure Creation: plt.figure(figsize=(10, 4)) creates a canvas that's 10 inches wide and 4 inches tall
  4. First Subplot: plt.subplot(1, 2, 1) means 1 row, 2 columns, and this is position 1 (left side)
  5. Second Subplot: plt.subplot(1, 2, 2) places the plot at position 2 (right side)
  6. Layout Adjustment: plt.tight_layout() automatically adjusts spacing to prevent overlapping
  7. Display: plt.show() renders the complete figure

üéì Professor explained:

"When we run this code, we get two subplots side by side. The left side shows the sine wave in blue color, and the right side shows the cosine wave in red color. The x-axis ranges from 0 to 10 (100 data points), and the y-axis automatically adjusts from -1 to +1 because all sine and cosine values lie between -1 and +1."

üí° Understanding np.linspace()

The function np.linspace(start, stop, num) generates num evenly spaced numbers between start and stop (inclusive). For example:

  • np.linspace(0, 10, 5) ‚Üí [0, 2.5, 5.0, 7.5, 10.0]
  • np.linspace(0, 10, 100) ‚Üí 100 points from 0 to 10

‚ö†Ô∏è Common Mistakes

  • Forgetting to call plt.figure() before creating subplots
  • Using wrong index numbers (remember: starts from 1, not 0)
  • Not calling plt.tight_layout(), which may cause overlapping labels
  • Placing plt.show() before all subplots are created

Using plt.subplots() ‚Äî Preferred Method

The plt.subplots() function (note the 's') is the modern and preferred way to create subplots. Unlike plt.subplot(), this function creates all subplots at once and returns both the figure and axes objects, giving you more control and cleaner code.

Why is plt.subplots() Preferred?

Feature plt.subplot() plt.subplots()
Creation Method Creates subplots one at a time Creates all subplots at once
Syntax More verbose, repetitive Cleaner, more concise
Access Method Current subplot is active Access via axes array (more flexible)
Indexing Starts from 1 Starts from 0 (Pythonic)
Control Less control over individual subplots Full control via axes objects

Syntax


fig, axs = plt.subplots(nrows, ncols, figsize=(width, height))

Returns:

  • fig: The figure object (entire canvas)
  • axs: Array of axes objects (individual subplots)

Complete Example: 2√ó2 Grid with Three Plots

üéì Professor explained:

"Now we'll use the preferred method. We have three datasets - y1 is linear (x), y2 is quadratic (x¬≤), and y3 is square root (‚àöx). We'll create a 2√ó2 grid, use three positions for plots, and leave one empty. The beauty here is that we can access each subplot using array indexing like axs[0,0], axs[0,1], etc."


# Step 1: Import libraries 
import matplotlib.pyplot as plt 
import numpy as np

# Step 2: Generate data 
x = np.linspace(0, 5, 100)      # 100 points from 0 to 5 
y1 = x                          # Linear: y = x 
y2 = x**2                       # Quadratic: y = x¬≤ 
y3 = np.sqrt(x)                 # Square root: y = ‚àöx

# Step 3: Create subplots in a 2√ó2 grid 
fig, axs = plt.subplots(2, 2, figsize=(10, 8))

# Step 4: Plot in position [0, 0] (top-left) 
axs[0, 0].plot(x, y1, 'g')      # 'g' = green 
axs[0, 0].set_title('Linear')

# Step 5: Plot in position [0, 1] (top-right) 
axs[0, 1].plot(x, y2, 'r')      # 'r' = red 
axs[0, 1].set_title('Quadratic')

# Step 6: Plot in position [1, 0] (bottom-left)    
axs[1, 0].plot(x, y3, 'b')      # 'b' = blue 
axs[1, 0].set_title('Square Root')

# Step 7: Hide the empty subplot [1, 1] (bottom-right) 
axs[1, 1].axis('off')

# Step 8: Add overall title 
fig.suptitle('Multiple Subplots Example', fontsize=16)

# Step 9: Adjust layout 
plt.tight_layout()

# Step 10: Display 
plt.show()

Understanding the Indexing System

In a 2√ó2 grid, the positions are accessed as follows:


[0, 0]	[0, 1]
[1, 0]	[1, 1]

üí° Array Indexing

When using plt.subplots(), the indexing is 0-based (Pythonic), unlike plt.subplot() which uses 1-based indexing. This makes it more consistent with Python's array indexing conventions.

Key Methods for Axes Objects

Method Purpose Example
axs[i, j].plot() Create a line plot axs[0, 0].plot(x, y)
axs[i, j].set_title() Set subplot title axs[0, 0].set_title('Linear')
axs[i, j].set_xlabel() Set x-axis label axs[0, 0].set_xlabel('X values')
axs[i, j].set_ylabel() Set y-axis label axs[0, 0].set_ylabel('Y values')
axs[i, j].axis('off') Hide axes and ticks axs[1, 1].axis('off')
fig.suptitle() Set main title for entire figure fig.suptitle('My Plots')

üéì Professor's tip:

"Notice how we use set_title() with axes objects instead of just title(). This is because we're working with specific subplot objects. Also, we can hide unused subplots using axis('off') - this removes both axes and makes the subplot area invisible, which is perfect when you have an odd number of plots in a grid."

Sharing Axes & Customizing Layout

One of the most powerful features of subplots is the ability to share axes between multiple plots. This is particularly useful when you want to compare data that exists on the same scale or time period.

Why Share Axes?

  • Easier Comparison: When X-axis is shared, vertical alignment helps identify patterns across different metrics
  • Space Efficiency: Removes redundant axis labels, making plots cleaner
  • Synchronized Zooming: In interactive plots, zooming on one subplot affects all shared-axis subplots
  • Visual Clarity: Clearly shows that all plots represent the same time period or scale

Sharing Axes Parameters

Parameter Values Description
sharex True / False Share X-axis across all subplots
sharey True / False Share Y-axis across all subplots

Complete Example: Trigonometric Functions with Shared X-axis

üéì Professor explained:

"Let's create three vertically stacked subplots showing sin(x), cos(x), and tan(x). Since all three functions are plotted over the same x range, we'll share the X-axis. This way, we can easily compare how these functions behave at the same x values. For tan(x), we need to be careful because it goes to infinity, so we'll limit the y-axis to -5 to +5."


# Step 1: Import libraries (assuming already imported) 
import matplotlib.pyplot as plt 
import numpy as np

# Step 2: Generate x data 
x = np.linspace(0, 10, 100)

# Step 3: Create 3 subplots vertically with shared x-axis 
fig, axs = plt.subplots(3, 1, figsize=(8, 8), sharex=True)

# Step 4: First subplot - Sine wave 
axs[0].plot(x, np.sin(x)) 
axs[0].set_title('sin(x)')

# Step 5: Second subplot - Cosine wave 
axs[1].plot(x, np.cos(x)) 
axs[1].set_title('cos(x)')

# Step 6: Third subplot - Tangent wave with limited y-axis 
axs[2].plot(x, np.tan(x)) 
axs[2].set_title('tan(x)') 
axs[2].set_ylim(-5, 5) # Limit y-axis to prevent infinity issues

# Step 7: Add overall figure title 
fig.suptitle('Trigonometric Functions', fontsize=16)

# Step 8: Adjust layout and display 
plt.tight_layout() plt.show()

Code Explanation

  1. Vertical Layout: plt.subplots(3, 1, ...) creates 3 rows and 1 column (vertical stack)
  2. Shared X-axis: sharex=True means all three subplots share the same x-axis scale
  3. 1D Indexing: Since we have only one column, we use axs[0], axs[1], axs[2] (not 2D indexing)
  4. Y-axis Limiting: set_ylim(-5, 5) restricts the tangent plot's y-axis to avoid infinity values

‚ö†Ô∏è The Tangent Function Problem

Problem: The tangent function, $\tan(x)$, has vertical asymptotes (approaches infinity) at $x = \frac{\pi}{2}, \frac{3\pi}{2}, \frac{5\pi}{2}, ...$

Solution: Use set_ylim() to restrict the y-axis range. This prevents the plot from becoming unreadable due to extreme values.

Professor's note: "Tangent varies from minus infinite to plus infinite. The -1 to +1 values may not be well represented graphically, so we limit the y-axis from -5 to +5. This is necessary to ensure all figures fit appropriately and remain readable."

Additional Customization Methods

Method Purpose Example
set_xlim(min, max) Set X-axis range axs[0].set_xlim(0, 10)
set_ylim(min, max) Set Y-axis range axs[2].set_ylim(-5, 5)
set_xlabel(label) Set X-axis label axs[2].set_xlabel('x values')
set_ylabel(label) Set Y-axis label axs[0].set_ylabel('Amplitude')
grid(True) Add grid lines axs[0].grid(True)
legend() Show legend axs[0].legend()

üí° Best Practices for Shared Axes

  • Time-series data: Always share X-axis when comparing different metrics over time
  • Same units: Share Y-axis when all subplots use the same measurement units
  • Label placement: With shared X-axis, only label the bottom subplot's X-axis
  • Title clarity: Use descriptive titles to distinguish between subplots
  • Consistent styling: Use similar colors and line styles for related plots

4. Creating Interactive Subplots using Plotly

Now that we understand both subplots and Plotly, let's combine them to create interactive subplots. Plotly's subplot functionality provides the same organizational benefits as Matplotlib subplots, but with the added advantage of interactivity.

Plotly Subplots Module

Plotly provides a dedicated module called plotly.subplots with a function make_subplots() that creates subplot layouts.


from plotly.subplots 
import make_subplots 
import plotly.graph_objects as go

make_subplots() Function

Basic Syntax


fig = make_subplots(rows=2, cols=2, 
subplot_titles=('Plot 1', 'Plot 2', 'Plot 3', 'Plot 4'))

Key Parameters

Parameter Description Example
rows Number of rows in the subplot grid rows=2
cols Number of columns in the subplot grid cols=3
subplot_titles Tuple of titles for each subplot ('Plot A', 'Plot B')
shared_xaxes Share X-axis across subplots shared_xaxes=True
shared_yaxes Share Y-axis across subplots shared_yaxes=True

horizontal_spacing Horizontal space between subplots (0 to 1) horizontal_spacing=0.1

Complete Example: Interactive Sine and Cosine Subplots


# Import required libraries 
from plotly.subplots 
import make_subplots 
import plotly.graph_objects as go 
import numpy as np

# Generate data 
x = np.linspace(0, 10, 100) 
y1 = np.sin(x) 
y2 = np.cos(x)

# Create subplots (1 row, 2 columns) 
fig = make_subplots(rows=1, cols=2, subplot_titles=('Sine Wave', 'Cosine Wave'))

# Add sine wave to first subplot (row=1, col=1) 
fig.add_trace( go.Scatter(x=x, y=y1, mode='lines', name='sin(x)', line=dict(color='blue')), row=1, col=1 )

# Add cosine wave to second subplot (row=1, col=2) 
fig.add_trace( go.Scatter(x=x, y=y2, mode='lines', name='cos(x)', line=dict(color='red')), row=1, col=2 )

# Update layout 
fig.update_layout( title_text='Interactive Trigonometric Functions', height=400, showlegend=True )

# Update x and y axis labels 
fig.update_xaxes(title_text='x', row=1, col=1) 
fig.update_xaxes(title_text='x', row=1, col=2) 
fig.update_yaxes(title_text='sin(x)', row=1, col=1) 
fig.update_yaxes(title_text='cos(x)', row=1, col=2)

# Display the figure 
fig.show()

Understanding the Code

  1. make_subplots(): Creates the subplot structure with 1 row and 2 columns
  2. add_trace(): Adds plots to specific subplot positions using row and col parameters
  3. go.Scatter(): Creates scatter/line plots (mode='lines' for line plots)
  4. update_layout(): Sets overall figure properties like title and height
  5. update_xaxes() / update_yaxes(): Updates axis labels for specific subplots

üí° Plotly Indexing

Unlike Matplotlib's plt.subplots() which uses 0-based indexing, Plotly's make_subplots() uses 1-based indexing. The first subplot is at row=1, col=1, not row=0, col=0.

Interactive Features

When you run the above code, the resulting plot will have several interactive features:

  • Hover: Move your mouse over the plot to see exact (x, y) values
  • Zoom: Click and drag to zoom into a specific region
  • Pan: After zooming, click and drag to move around
  • Reset: Double-click to reset to original view

Q1: What is the main function used to create subplots in Plotly?

Answer: The make_subplots() function from the plotly.subplots module is used to create subplot layouts in Plotly.

Q2: How does indexing differ between Matplotlib and Plotly subplots?

Answer: Matplotlib uses 0-based indexing (axs[0, 0] for first subplot), while Plotly uses 1-based indexing (row=1, col=1 for first subplot).

Q3: What are three interactive features available by default in Plotly plots?

Answer: (1) Hover to see exact values, (2) Zoom by clicking and dragging, and (3) Pan to move around after zooming. Additional features include reset (double-click) and downloading as PNG.

Q4: When should you use Matplotlib over Plotly?

Answer: Use Matplotlib when you need static images for academic papers or printed reports, when file size is a concern, or when working offline without web browser access.

Q5: How do you add a plot to a specific subplot position in Plotly?

Answer: Use fig.add_trace(plot_object, row=row_number, col=col_number) to add a plot to a specific position in the subplot grid.
