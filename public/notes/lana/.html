Understanding the Concept

In Linux, every file or directory has three sets of permissions — for the owner, group, and others. Each permission (Read, Write, Execute) can be represented in binary and converted into octal numbers.

Permission Binary Octal Value
Read (r) 100 4
Write (w) 010 2
Execute (x) 001 1

Each set of permissions is represented by three bits, and combining these gives an octal number (base 8). For example, rw-r--r-- corresponds to 644 in octal.

Question 1: Executable Script

Scenario: You have created a shell script and need to set permissions such that:

  • Owner can read, write, and execute
  • Group can read and execute
  • Others can only execute
User Permissions Binary Octal
Owner r + w + x 111 7
Group r + x 101 5
Others x 001 1

Final Octal Representation: 751

Linux Command:

chmod 751 script.sh

Question 2: Restricted Access File

Scenario: You have a configuration file and need to set permissions such that:

  • Owner can read and write
  • Group has no permissions
  • Others can only read
User Permissions Binary Octal
Owner r + w 110 6
Group --- 000 0
Others r 100 4

Final Octal Representation: 604

Linux Command:

chmod 604 config.txt

Conclusion

File permissions in Linux are a direct application of the octal number system. Each permission type (read, write, execute) corresponds to specific bits in binary, and combining them gives octal codes that can be efficiently applied using the chmod command.
